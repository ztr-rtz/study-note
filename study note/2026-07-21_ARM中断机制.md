# ARM 中断机制（IMX6ULL）

## 一、中断处理全流程

中断发生时，CPU不是一上来就跳转，而是按固定步骤走完才算一次完整中断：

| 步骤 | 做了什么 | 一句话总结 |
|------|----------|-----------|
| 1 | 中断源发出中断请求 | 外设说"我有事要汇报" |
| 2 | CPU检查是否响应 + 是否被屏蔽 | 总开关和子开关都得打开 |
| 3 | 比较中断优先级 | 同时多个请求，先处理急的 |
| 4 | 保护现场 | 把当前寄存器压栈存好 |
| 5 | 执行中断服务函数 | 真正干活的部分 |
| 6 | 恢复现场 | 从栈里把寄存器弹回来，继续原来的活 |

核心逻辑：**中断不是随时都能进的**。第2步的"总开关"是CPSR的I位（IRQ使能），"子开关"是外设自己的中断使能位。两个开关都打开才算放行。

## 二、IMX6ULL 中断信号路径

信号怎么从按键传到CPU核？不是直连，中间经过好几层：

```
外设(GPIO1_18按键) → GPIO控制器 → GIC(中断控制器) → ARM内核
```

完整配置和响应流程：

1. 开启CPU总中断（`cpsie i`）
2. 切换工作模式 → 进入IRQ模式（自动）
3. 使能外设中断（GPIO1_18引脚中断使能）
4. 设置中断优先级
5. GIC接收信号 → 跳转到异常向量表 `irq_handler`
6. 在irq_handler里调用 `system_irqhandler(R0)`，R0存的是中断号
7. 处理完毕 → 恢复现场（工作模式也跟着恢复回来）

## 三、GIC：通用中断控制器

IMX6ULL用的是GIC-400（ARM架构标配），负责接收所有外设中断信号、排序优先级、再分发给内核。

### GIC版本演进

| 版本 | 适用处理器 | 说明 |
|------|-----------|------|
| V1 | Cortex-A5 | 早期 |
| V2 | Cortex-A7（32位） | IMX6ULL就在这里 |
| V3-V4 | Cortex-A53（64位） | 手机芯片 |
| V5 | 数百核处理器 | 服务器级算力 |

### 中断分类

GIC一共管160个中断号（0-159），分三类：

| 类型 | 编号范围 | 来源 | 例子 |
|------|---------|------|------|
| SGI（软中断） | 0-15 | 软件产生 | 核间通信、触发IPI |
| PPI（私有外设中断） | 16-31 | 核间通信 | 每个核私有的定时器中断 |
| SPI（共享外设中断） | 32-159 | 外设产生 | GPIO按键、UART接收 |

IMX6ULL实际支持160个中断（编号0-159），不是GIC理论上的1020。

### GIC关键寄存器

| 寄存器 | 地址偏移 | 方向 | 作用 |
|--------|---------|------|------|
| C_IAR | 0x200C | 只读 | 中断确认寄存器，读取后返回当前中断号 |
| C_EOIR | 0x2010 | 只写 | 中断结束寄存器，写完通知GIC"这个中断我处理完了" |

处理流程里必须做的事：先读C_IAR拿中断号，处理完再写C_EOIR。不写EOIR的话GIC不会释放这个中断号，后续同编号中断就进不来。

## 四、CP15协处理器

CP15是ARM的"系统配置管家"，中断相关操作离不开它。

### 两条核心指令

- **mrc**：从cp15寄存器读数据到ARM通用寄存器
- **mcr**：把ARM通用寄存器数据写入cp15寄存器

格式：`MCR<c> <coproc>, <opc1>, <Rt>, <CRn>, <CRm>{, <opc2>}`

### 中断相关的cp15操作

| 操作 | 指令 | 做了什么 |
|------|------|----------|
| 读取SCTLR | `mrc p15, 0, r0, c1, c0, 0` | 拿系统控制寄存器 |
| 关闭高向量 | `bic r0, r0, #(1<<13)` | SCTLR的bit13=0 → 低向量地址 |
| 开启分支预测 | `orr r0, r0, #(1<<12)` | SCTLR的bit12=1 → 分支预测使能 |
| 写回SCTLR | `mcr p15, 0, r0, c1, c0, 0` | 配置生效 |
| 设置异常向量表基址 | `mcr p15, 0, r0, c12, c0, 0` | VBAR寄存器，指定向量表起始地址 |
| 读取GIC基地址 | `mrc p15, 4, r1, c15, c0, 0` | CBAR寄存器 → GIC基地址 |

最后一行特别重要：GIC不是固定地址的，得从CBAR读出来才能定位GIC寄存器。

## 五、异常向量表

ARM处理器遇到异常（中断、未定义指令、复位等）时，不是漫无目的地跳，而是去一张固定表格里找入口。

| 地址 | 指令 | 异常类型 |
|------|------|----------|
| 0x00 | `b reset_handler` | 复位 |
| 0x04 | `b undefine_handler` | 未定义指令 |
| 0x08 | `b svc_handler` | 软中断(SWI) |
| 0x0C | `b prefetch_handler` | 预取指令异常 |
| 0x10 | `b abort_handler` | 数据访问异常 |
| 0x14 | 保留 | — |
| 0x18 | `b irq_handler` | IRQ中断 |
| 0x1C | `b fiq_handler` | FIQ快速中断 |

中断向量 vs 异常向量的区别：
- **中断向量**：标号，通过它找到中断服务函数
- **异常向量**：跳转到异常服务函数指令的地址
- **中断向量表**：数组，存放中断服务函数入口地址
- **异常向量表**：数组，存放跳转指令（`b xxx_handler`）

用C语言类比：`void (*p[8])(void) = {reset, undef, svc, prefetch, abort, NULL, irq, fiq};`

## 六、CPSR中断控制

CPSR（当前程序状态寄存器）里有两个中断控制位：

| 操作 | 指令 | 效果 |
|------|------|------|
| 使能IRQ | `cpsie i` | I位清零，CPU开始响应IRQ |
| 禁止IRQ | `cpsid i` | I位置1，CPU屏蔽IRQ |
| 使能FIQ | `cpsie f` | F位清零，CPU开始响应FIQ |
| 禁止FIQ | `cpsid f` | F位置1，CPU屏蔽FIQ |

`cps`指令本身就是模式切换指令，`ie`/`id`是使能/禁止后缀，`i`对应IRQ，`f`对应FIQ。

## 七、GPIO按键中断配置步骤（Key0 → GPIO1_18）

用IMX6ULL上的Key0按键做中断实验，完整配置：

1. 配置引脚复用功能（iomuxc，把GPIO1_18设为GPIO模式）
2. 配置引脚电气属性（padctrl，驱动能力、上下拉等）
3. 配置输入方向（GPIO1_GDIR，bit18=0 → 输入）
4. 配置中断触发方式（边沿/电平触发）
5. 使能GPIO1_18引脚中断（GPIO1_IMR，bit18=1）
6. 初始化GIC：`GIC_Init()`
7. 使能GPIO1_18中断号：`GIC_EnableIRQ(GPIO1_Combined_16_31_IRQn)`
8. 设置优先级（GIC优先级寄存器）

然后等按键按下，中断进来，走前面说的完整流程。

## 八、速查表

| 概念 | 要点 |
|------|------|
| 中断6步流程 | 请求→检查→优先级→保护现场→服务函数→恢复现场 |
| 信号路径 | 外设→GPIO→GIC→内核 |
| GIC三类中断 | SGI(0-15) / PPI(16-31) / SPI(32-159) |
| GIC必做 | 读IAR拿号 → 处理 → 写EOIR释放号 |
| cp15读GIC基址 | `mrc p15, 4, r1, c15, c0, 0`（CBAR） |
| CPSR开关 | `cpsie i`开IRQ / `cpsid i`关IRQ |
| 异常向量表 | 8个入口，IRQ在0x18 |
| SPSR | 异常发生时自动保存CPSR，返回时恢复 |

## 九、易错提醒

1. **EOIR必须写**：不写C_EOIR，GIC不会释放中断号，同号中断永远进不来
2. **两个开关都要开**：CPU总中断（cpsie i）+ 外设子开关（GPIO IMR位），只开一个没用
3. **GIC基地址不是硬编码**：必须从CBAR读，不同芯片GIC位置不同
4. **保护现场和恢复现场必须对称**：压了几个寄存器就弹几个，顺序也要对
5. **向量表基址要用mcr写入VBAR**：不是默认就指向你写的那个表，得手动设置
6. **SPI编号从32开始**：不是从0，GPIO中断属于SPI类

## 十、问答：中断核心概念

### Q1：什么是GIC？

**GIC（Generic Interrupt Controller，通用中断控制器）** 是ARM公司设计的中断管理硬件模块。它卡在外设和CPU核之间，干三件事：

- **接收**：收集所有外设发来的中断信号
- **仲裁**：按优先级排序，决定哪个中断先送给CPU
- **分发**：把选出来的中断发给对应的CPU核处理

没有GIC的话，CPU要自己挨个轮询外设"你有事没？"，效率极低。有了GIC，外设主动通知，CPU只在需要的时候才被打断。

IMX6ULL用的是GIC-400，属于GIC V2版本，面向Cortex-A7这类32位处理器。GIC-400不是ARM自己流片的芯片，而是一个IP核（RTL代码），芯片厂商（如NXP）把它集成进IMX6ULL里。

### Q2：什么是协处理器？ARM Cortex-A7配备了几个协处理器？

**协处理器**是挂在ARM核旁边的辅助处理单元，用来扩展CPU的能力——主核做通用计算，协处理器干特定领域的事。

ARm架构最大支持 **16个协处理器**，编号cp0-cp15，每个协处理器对应ARM指令集编码中4位的协处理器编号字段。

**Cortex-A7实际用到的协处理器：**

| 编号 | 用途 | 是否配备 |
|------|------|---------|
| cp0-cp9 | 保留/未使用 | 可选 |
| cp10 | VFP浮点运算控制 | ✅ 有（如果使能NEON/VFP） |
| cp11 | VFP浮点运算数据 | ✅ 有（与cp10配对使用） |
| cp14 | Debug调试控制 | ✅ 有 |
| cp15 | 系统控制（cache/MMU/中断/TCM等） | ✅ **必须有** |

cp15是最核心的一个。中断相关的所有配置——读取GIC基址、设置异常向量表地址、开关MMU/cache——全走cp15。

如果程序访问了一个不存在的协处理器，ARM核会触发未定义指令异常（Undefined Instruction Exception），跳转到异常向量表0x04地址。

协处理器指令主要有三类：

| 类型 | 指令 | 作用 |
|------|------|------|
| 数据处理 | CDP | 初始化协处理器内部运算 |
| 寄存器传送 | MRC / MCR | ARM寄存器 ↔ 协处理器寄存器 |
| 存储器传送 | LDC / STC | 内存 ↔ 协处理器寄存器 |

中断编程最常用的是MRC/MCR：读cp15配置、写cp15配置。

### Q3：为什么要将异常向量表重映射？如何重映射？

**为什么要重映射：**

ARM默认的异常向量表在地址 **0x00000000**。但问题来了——芯片上电后，0x00000000这个位置通常被Boot ROM占用（只读），没法在上面放自己的向量表。

还有另一个问题：IMX6ULL启动过程里，程序最终要拷贝到DDR（比如0x80000000）去跑。如果向量表还在0x00000000，调用关系就断了——异常发生时CPU还是往0x00000000跳，而不是跳到你DDR里写的向量表。

所以需要把向量表"搬"到你代码实际在的位置。这就是重映射。

**如何重映射：**

走cp15的VBAR寄存器（Vector Base Address Register）：

```assembly
ldr r0, =0x80000000          @ 假设代码在DDR的0x80000000起始
mcr p15, 0, r0, c12, c0, 0   @ 把VBAR设为0x80000000
```

这一句执行完，异常发生时CPU就改道往VBAR指向的地址偏移去找向量入口，而不是死磕0x00000000。

具体跳转地址 = VBAR基址 + 异常类型偏移量。比如IRQ发生时，CPU跳转到 `VBAR + 0x18`。

### Q4：GIC支持多少个中断，分为哪几类？

GIC-400理论上支持最多 **1020个中断**（编号0-1019），分三类：

| 类型 | 编号范围 | 数量 | 来源 | 典型场景 |
|------|---------|------|------|----------|
| SGI（软件生成中断） | 0-15 | 16个 | 软件写寄存器触发 | 核间通信、多核调度 |
| PPI（私有外设中断） | 16-31 | 16个 | 每个核私有的外设 | 核私有定时器、性能计数器 |
| SPI（共享外设中断） | 32-1019 | 988个 | 片上外设 | GPIO、UART、I2C、SPI、定时器等 |

三类中断的关键区别：
- **SGI**不是外设产生的，是软件故意触发的，专门用来让不同CPU核互相通知
- **PPI**每个核有自己的独立一份，互不干扰
- **SPI**所有核共用同一份，可以被路由到任意一个或多个核

### Q5：IMX6ULL支持多少个中断？

IMX6ULL实际实现了 **160个中断**，编号 0-159。虽然GIC-400理论上能管1020个，但NXP只用了前160个。

这160个中断的分布：

| 类型 | 编号 | 数量 |
|------|------|------|
| SGI | 0-15 | 16个 |
| PPI | 16-31 | 16个 |
| SPI | 32-159 | 128个 |

常用的SPI中断号举例：

| 中断源 | 中断号 | 说明 |
|--------|--------|------|
| GPIO1_Combined_0_15 | 98 | GPIO1的0-15引脚合用一个中断线 |
| GPIO1_Combined_16_31 | 99 | GPIO1_16-31合用一个，Key0(GPIO1_18)走这里 |
| UART1 | 58 | 串口1收发中断 |
| GPT1 | 87 | 通用定时器1 |

注意：IMX6ULL是单核Cortex-A7芯片，所以SGI虽然支持但实际用不上核间通信的典型场景。GPIO是按组（16个引脚合一个中断号），进中断后再读相关寄存器判断具体是哪个引脚触发的。
