# 多任务并发

> 整理自：Linux应用软件编程 · 第二章
> 核心：进程 vs 线程、并发控制、IPC 通信

---

## 一、进程基础

### 1. 概念区分

| 维度 | 程序 | 进程 |
|------|------|------|
| 状态 | 静态 | 动态 |
| 位置 | 硬盘 | 内存 |
| 资源 | 不消耗 | 消耗 CPU + 内存 |
| 数量 | 一份 | 可创建多个 |

进程是**操作系统资源分配的最小单位**，执行期间需要 0~4G 虚拟内存空间。

### 2. 进程状态（Linux 七态）

```
        ┌──────────┐
   ┌───▶│ 运行态 R │─────┐
   │    └──────────┘     │
   │ 调度/时间片到        │ 被调度
   │    ┌──────────┐     │
   └───│ 就绪态 R │◀────┘
        └──────────┘
              │ 等待资源
        ┌──────────┐
        │可唤醒等待 S│
        └──────────┘
              │ 信号到达
        ┌──────────┐
        │ 运行态 R │
        └──────────┘
```

| 状态 | 标记 | 说明 |
|------|------|------|
| 运行态 | R | 正被 CPU 调度执行 |
| 就绪态 | R | 只缺 CPU，万事俱备 |
| 可唤醒等待 | S | 阻塞等资源，可被信号唤醒 |
| 不可唤醒等待 | D | 不想被打断，如磁盘 I/O |
| 暂停态 | T | 被暂停 |
| 僵尸态 | Z | 已结束但父进程未回收 |
| 结束态 | X | 已结束且资源已回收 |

### 3. 常用进程命令

| 命令 | 作用 |
|------|------|
| `ps -aux` | 查看进程详情 |
| `ps -ef` | 查看父进程 PID |
| `pstree -sp PID` | 查看进程族谱 |
| `top` | 动态看 CPU/内存占用 |
| `kill -9 PID` | 强制结束进程 |
| `kill -19 PID` | 暂停进程 |
| `kill -18 PID` | 继续运行 |
| `jobs` | 看后台进程 |
| `fg 编号` | 后台进程切前台 |

## 二、进程编程

### 1. fork 创建子进程

```c
pid_t fork(void);
// 父进程返回子进程 PID；子进程返回 0；失败返回 -1
```

**拷贝策略**：
- 子进程完全拷贝父进程 0~3G 用户空间（文本区、数据区、堆区、栈区）
- 拷贝 PCB 部分内容，但 **PID 不拷贝**
- 父子用户空间独立，数据不共享

**相关函数**：
```c
pid_t getpid(void);   // 获取自己 PID
pid_t getppid(void);  // 获取父进程 PID
```

### 2. 进程退出

| 方式 | 特性 |
|------|------|
| `return` | 主函数正常退出 |
| `exit()` | 库函数，退出前**刷新缓冲区** |
| `_exit()` / `_Exit()` | 系统调用，退出前**不刷新缓冲区** |
| 信号 | 被动终止，如 `kill -9` |

### 3. 回收资源 — wait / waitpid

```c
pid_t wait(int *wstatus);       // 阻塞等待回收任意僵尸子进程
pid_t waitpid(pid_t pid, int *status, int options);
```

**waitpid 参数**：
- `pid = -1`：回收任意子进程
- `pid = 0`：回收同组子进程
- `pid > 0`：回收指定 PID
- `options = 0`：阻塞；`WNOHANG`：非阻塞

**状态判断宏**：
```c
WIFEXITED(wstatus)     // 是否正常结束
WEXITSTATUS(wstatus)   // 取 exit 状态码
WIFSIGNALED(wstatus)   // 是否被信号终止
WTERMSIG(wstatus)      // 若是信号终止，取信号编号
```

### 4. 孤儿进程与僵尸进程

- **僵尸进程**：子进程结束，父进程未回收，残留 PCB
- **孤儿进程**：父进程先结束，子进程被 init 收养（PID 1），最终由系统回收

### 5. exec 函数族

```c
int execl(const char *path, const char *arg, ..., NULL);   // 指定路径
int execlp(const char *file, const char *arg, ..., NULL);  // PATH 中找
int execv(const char *path, char *const argv[]);           // 参数用数组
```

| 后缀 | 含义 |
|------|------|
| `l` | list，参数逐个传 |
| `v` | vector，参数用数组 |
| `p` | path，自动从 PATH 环境变量查找 |
| `e` | env，自定义环境变量 |

原理：用新程序的文本段替换当前进程的文本段，**PID 不变**。

## 三、线程基础

### 1. 线程 vs 进程

| 维度 | 进程 | 线程 |
|------|------|------|
| 本质 | 正在执行的程序 | 轻量级进程 |
| 资源分配 | 最小单位 | 调度最小单位 |
| 内存空间 | 独立 0~4G | 栈独立（默认 8M），堆/数据/文本共享 |
| 创建/切换效率 | 低 | 高 |
| 安全性 | 高（空间隔离） | 低（共享内存） |
| 通信方式 | IPC（复杂） | 全局变量（简单） |
| 并发量 | 少 | 多 |

### 2. 线程编程接口

```c
int pthread_create(pthread_t *thread, const pthread_attr_t *attr,
                 void *(*start_routine)(void *), void *arg);
// 编译链接必须加 -lpthread

void pthread_exit(void *retval);           // 线程结束
pthread_t pthread_self(void);              // 获取线程 ID
int pthread_join(pthread_t thread, void **retval); // 阻塞回收线程
```

### 3. 分离属性

```c
int pthread_detach(pthread_t thread);
```
- 分离线程：结束时由操作系统自动回收，无需其他线程 join
- 默认属性：**非分离**，需要被 join 回收

## 四、线程同步与互斥

### 1. 互斥锁（解决资源竞争）

```c
pthread_mutex_t mutex;                        // 定义
pthread_mutex_init(&mutex, NULL);              // 初始化
pthread_mutex_lock(&mutex);                    // 加锁（阻塞）
pthread_mutex_trylock(&mutex);                 // 加锁（非阻塞）
pthread_mutex_unlock(&mutex);                  // 解锁
pthread_mutex_destroy(&mutex);                 // 销毁
```

### 2. 信号量（实现线程同步）

信号量 = 资源计数器。>0 时可直接申请；=0 时阻塞等待。

```c
sem_t sem;
sem_init(&sem, 0, 1);     // 0:线程间  1:初始资源数
sem_wait(&sem);             // P操作，申请资源，资源-1
sem_post(&sem);             // V操作，释放资源，资源+1
sem_destroy(&sem);
```

### 3. 死锁

**定义**：多个线程互相持有对方想要的资源，永久等待。

**四个必要条件**（全中 = 死锁，破任一 = 解除）：
1. **互斥**：资源同一时刻只能被一个任务占用
2. **请求与保持**：拿着资源不放，还去申请新资源
3. **不剥夺**：资源只能由持有者主动释放
4. **循环等待**：形成头尾相接的等待环

**常见现象**：
- 忘记释放锁
- 重复加锁
- A 拿锁1要锁2，B 拿锁2要锁1

**解决方法**：
- 锁成对出现（加锁必解锁）
- 统一加锁顺序
- 使用非阻塞锁，拿不到就释放自己已有的资源

## 五、进程间通信（IPC）

同一主机通信方式：

| 方式 | 特点 | 适用场景 |
|------|------|----------|
| 无名管道 | 单向，亲缘进程间 | 父子进程通信 |
| 有名管道 | 单向，任意进程 | 无亲缘关系进程 |
| 信号 | 异步通知，软中断 | 事件通知 |
| 消息队列 | 按类型收发，可优先级 | 结构化数据传递 |
| 共享内存 | **效率最高**，内存映射 | 大数据量共享 |
| 信号量集 | 进程间同步 | 资源计数 |

### 1. 无名管道 pipe

```c
int pipe(int pipefd[2]);
// pipefd[0] 读端  pipefd[1] 写端
```

特点：
- 单向通道，读端写端不能交换
- 数据先进先出（FIFO）
- 管道大小默认 64K（65536 字节）
- 读走即剪切，不保留

管道行为：
| 场景 | 结果 |
|------|------|
| 读写端都存在，读空管道 | 读阻塞 |
| 读写端都存在，写满管道 | 写阻塞 |
| 读端关闭，继续写 | 管道破裂（异常） |
| 写端关闭，继续读 | 有数据读数据，无数据返回 0 |

### 2. 有名管道 mkfifo

```c
int mkfifo(const char *pathname, mode_t mode);
// 创建一个管道文件（p类型），之后用 open/read/write/close 操作
```

### 3. 信号 signal

```c
typedef void (*sighandler_t)(int);
sighandler_t signal(int signum, sighandler_t handler);
// handler: SIG_IGN 忽略  /  SIG_DFL 默认  /  自定义函数 捕获
```

常用信号速查：
| 信号 | 编号 | 触发方式 | 作用 |
|------|------|----------|------|
| SIGINT | 2 | `Ctrl + C` | 中断进程 |
| SIGQUIT | 3 | `Ctrl + \` | 退出进程 |
| SIGKILL | 9 | `kill -9` | 强制结束（不可捕获/忽略） |
| SIGSEGV | 11 | 段错误 | 非法内存访问 |
| SIGPIPE | 13 | 管道破裂 | 写已关闭读端的管道 |
| SIGALRM | 14 | `alarm()` | 定时器到期 |
| SIGCHLD | 17 | 子进程结束 | 发给父进程，通知回收 |
| SIGSTOP | 19 | `kill -19` | 强制暂停（不可捕获/忽略） |
| SIGCONT | 18 | `kill -18` | 继续运行 |

```c
unsigned int alarm(unsigned int seconds);   // 设置闹钟，到时发 SIGALRM
int pause(void);                             // 挂起进程，等信号唤醒
```

注意：信号处理函数要**快、短、小**，避免阻塞/休眠/长耗时操作。

### 4. 消息队列

```c
key_t ftok(const char *pathname, int proj_id);  // 获取 IPC key
int msgget(key_t key, int msgflg);              // 创建/获取队列 ID
int msgsnd(int msqid, const void *msgp, size_t msgsz, int msgflg); // 发送
ssize_t msgrcv(int msqid, void *msgp, size_t msgsz, long msgtyp, int msgflg); // 接收
int msgctl(int msqid, int cmd, struct msqid_ds *buf); // 控制（删除）
```

消息体结构：
```c
struct msgbuf {
    long mtype;      // 消息类型，必须 > 0
    char mtext[...]; // 正文数据
};
```

msgtyp 取值：
- `0`：按顺序收
- `>0`：收指定 mtype 的消息
- `<0`：收绝对值小于等于 |msgtyp| 的消息

### 5. 共享内存（效率最高的 IPC）

```c
int shmget(key_t key, size_t size, int shmflg);           // 创建共享内存
void *shmat(int shmid, const void *shmaddr, int shmflg);  // 建立映射
int shmdt(const void *shmaddr);                           // 解除映射
int shmctl(int shmid, int cmd, struct shmid_ds *buf);   // 删除等操作
```

size 向上取整到 PAGE_SIZE（4096）的整数倍。

### 6. IPC 管理命令

```bash
ipcs -a          # 查看所有 IPC 对象（共享内存、信号量集、消息队列）
ipcrm -m id     # 删除共享内存
ipcrm -s id     # 删除信号量集
ipcrm -q id     # 删除消息队列
```

## 六、易错提醒

1. `fork` 后父子进程独立运行，不要假设执行顺序
2. 子进程退出务必回收，否则变僵尸进程
3. `waitpid` 非阻塞模式（`WNOHANG`）要配合轮询使用
4. 线程中不能用 `exit()`，会干掉整个进程；用 `pthread_exit()`
5. 主线程不能先于子线程退出，否则子线程会被强制终止
6. 互斥锁必须成对出现，加锁后所有分支都要解锁
7. 死锁最常见原因：加锁顺序不一致 + 忘记释放
8. 管道读端关闭后继续写，触发 `SIGPIPE` 信号
9. 信号处理函数里不要做阻塞操作，不要调不可重入函数
10. 共享内存本身不互斥，需配合信号量或互斥锁使用
11. `ftok` 产生 key 时，pathname 必须是已存在的文件
