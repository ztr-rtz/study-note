# 定时器 - EPIT 与 GPT

> 来源：`定时器.pdf`（单页笔记，含定时器基础概念 + Modbus/RS485/UART 杂记）
> 用户提问：①什么是 EPIT、什么是 GPT ②位数和计数方向

---

## 一、PDF 原文要点速览

笔记里画了这些基础概念：

- **16 位计数器范围**：`0x0000 ~ 0xFFFF`（共 65536 个计数）
- **32 位计数器范围**：`0x00000000 ~ 0xFFFFFFFF`（共约 42.9 亿个计数）
- **输入捕获**：从某个引脚上捕获两次电平跳变之间的时间间隔（抓方波）
- **输出比较**：跟计数寄存器里的值比对，周期性地输出方波
- **计数溢出**：`old_counter → new_counter`，从 `0x0` 一路计到 `0xFFFFFFFF` 后溢出回卷
- **Linux 多任务切片**：A / B / C 三个任务各占 0.5ms

> 注意：PDF 里没明写 EPIT / GPT 字样，下面两个模块的信息来自 IMX6ULL 参考手册 + 上课笔记。

## 二、EPIT 是什么？

**EPIT** = **Enhanced Periodic Interrupt Timer**（增强型周期中断定时器）

顾名思义，它的核心用途就是"周期性地产生中断"，适合做精准定时、OS 心跳、按键消抖这种场景。

主要特征：
- 一组 32 位计数器 + 比较寄存器
- 计到比较值就触发中断，可设自动重装载 → 周期触发不用 CPU 干预
- 支持 set-and-forget 模式（写一次比较值后硬件自动 reload）
- 时钟源可选 IPG clock / IPG clock / High-frequency reference clock

应用场景：周期性任务调度、毫秒/微秒级延时、心跳、喂狗前的定时等。

---

## 三、GPT 是什么？

**GPT** = **General Purpose Timer**（通用定时器）

功能比 EPIT 丰富得多，除了定时，还能干"输入捕获"和"输出比较"的活，正好对应笔记里写的两个概念。

主要特征：
- 32 位向上计数器
- 支持三种工作模式：Restart（到比较值自动清零重数） / Free-Run（一直数到溢出再回 0） / Wait（计数到比较值就停）
- 3 路 **输出比较**通道（OC1/OC2/OC3），可输出 PWM
- 2 路 **输入捕获**通道（IC1/IC2），可测方波频率/占空比
- 时钟源可选 IPG clock / High-frequency reference clock / External clock / Low-frequency reference clock

应用场景：PWM 输出、电机控制、测速（配合编码器）、超声波测距、红外解码、超声波/红外/舵机控制等。

## 四、位数和计数方向（核心问题）

| 特性 | EPIT | GPT |
|---|---|---|
| 位数 | **32 位** | **32 位** |
| 计数方向 | **向下计数**（从装载值往 0 减） | **向上计数**（从 0 往 0xFFFFFFFF 加） |
| 计数范围 | 0xFFFFFFFF → 0x00000000 | 0x00000000 → 0xFFFFFFFF |
| 触发中断 | 减到比较值触发 | 加到比较值触发 |
| 输入捕获 | ❌ 不支持 | ✅ 支持 2 路 |
| 输出比较 | ❌ 不支持 | ✅ 支持 3 路 |
| PWM 输出 | ❌ 不直接支持 | ✅ 直接支持 |
| 主要用途 | 周期中断、心跳、延时 | 测频、测脉宽、PWM、电机控制 |

**一句话记牢**：
- **EPIT**：32 位、向下、纯定时中断专用
- **GPT**：32 位、向上、定时 + 输入捕获 + 输出比较全能

## 五、为什么 PDF 里画"计数溢出"那个图？

图里画的是 `0x0` 走到 `0xFFFFFFFF` 再回卷的半圆，对应的是**向上计数**溢出——也就是 **GPT** 的行为（free-run 模式下数到顶自动回到 0 重数）。

EPIT 是向下计数，方向相反：从装载值开始往下减，减到比较值或 0 就触发中断/重装载。

PDF 上还写了 `old_counter` 和 `new_counter`，这是用来算两次捕获之间计数器走了多少步的，对应 GPT 的**输入捕获**用法（记录方波两次跳变时刻的计数值，做差再乘以计数周期就是时间间隔）。

## 六、速查表

| 项目 | EPIT | GPT |
|---|---|---|
| 全称 | Enhanced Periodic Interrupt Timer | General Purpose Timer |
| 位宽 | 32 位 | 32 位 |
| 计数方向 | **向下计数** | **向上计数** |
| 装载/比较寄存器 | EPIT_CMPR / EPIT_LR | GPT_OCR1/2/3 / GPT_IR |
| 输入捕获 | 无 | 2 路（IC1、IC2） |
| 输出比较 | 无 | 3 路（OC1、OC2、OC3） |
| 计数模式 | set-and-forget | Restart / Free-Run / Wait |
| 典型用途 | 周期性中断、心跳 | PWM、测频、测速、编码器 |
| 工作位 | 32 位定时器里的"专用选手" | 32 位定时器里的"全能选手" |

## 七、易错提醒

1. **方向别记反**——EPIT 向下、GPT 向上，是 IMX6ULL 上的固定设计，面试/考试常考。
2. **位数都是 32 位**，虽然 STM32 上有 16 位定时器，但 IMX6ULL 的 EPIT/GPT 都是 32 位的。
3. **EPIT 没有捕获/比较通道**，需要 PWM 或测频时必须上 GPT。
4. **计数周期计算公式**：`T = (compare + 1) / clk_src`，向上向下都适用，只是比较值含义不同。
5. PDF 里画的 `0x0 → 0xFFFFFFFF` 回卷是 GPT 的典型工作场景，别误以为是 EPIT。
