# 文件编程（IO）

## 1. 为什么要学文件操作

- 数据掉电不丢，必须写到硬盘（外存）才行
- Linux哲学：一切皆文件——硬盘、鼠标、键盘、显示器、普通文件都当文件处理

## 2. Linux七种文件类型

`ls -l`（或 `ll`）看文件详情，首字符标识类型：

| 标识 | 类型 | 说明 | 操作方式 |
|------|------|------|----------|
| b | 块设备 | 硬盘 | 文件IO |
| c | 字符设备 | 鼠标、键盘、传感器 | 文件IO |
| d | 目录 | 文件夹 | 目录IO |
| - | 普通文件 | .c .h .jpg .txt 等 | 文件IO / 标准IO |
| l | 链接 | 类似快捷方式 | 看链接项 |
| s | 套接字 | 网络通信 | 文件IO |
| p | 管道 | 进程间通信 | 文件IO |

权限字段解读：`rwx rwx r-x` → 本人 / 同组 / 其他人，读写执行

## 3. 两种文件操作方式

| 对比项 | 标准IO | 文件IO |
|--------|--------|--------|
| 来源 | C标准库（库函数） | Linux内核（系统调用） |
| 代表 | printf, scanf | open, read, write |
| 面向 | FILE* 流指针 | fd 文件描述符 |
| 缓冲 | 有缓冲区 | 无缓冲 |
| 可移植 | 强，跨平台 | 弱，仅Linux |
| 适用 | 普通文件读写 | 硬件设备、管道、套接字 |

操作三步：打开 → 读写 → 关闭

## 4. 标准IO函数

### 4.1 fopen / fclose

```c
FILE *fopen(const char *pathname, const char *mode);
int fclose(FILE *stream);
```

打开方式一览：

| mode | 含义 | 文件不存在时 |
|------|------|-------------|
| "r" | 只读 | 必须存在 |
| "r+" | 读写 | 必须存在 |
| "w" | 只写 | 清空或创建 |
| "w+" | 读写 | 清空或创建 |
| "a" | 追加写 | 创建 |
| "a+" | 读+追加写 | 创建 |

文件流：读写数据时的字节流，FILE* 就是流指针，标准IO操作都面向它。

fopen失败返回NULL。

### 4.2 fputc / fgetc

```c
int fputc(int c, FILE *stream);  // 写一个字符，成功返回ASCII值，失败EOF
int fgetc(FILE *stream);         // 读一个字符，成功返回ASCII值，失败/末尾EOF
```

三个预打开的流：

| 流 | 设备 |
|----|------|
| stdin | 键盘 |
| stdout | 显示屏 |
| stderr | 显示屏 |

等价关系：`getchar()` = `fgetc(stdin)`；`putchar(c)` = `fputc(c, stdout)`

### 4.3 字符拷贝示例

```c
int copy_file(const char *srcname, const char *dstname) {
    FILE *fpsrc = fopen(srcname, "r");
    FILE *fpdst = fopen(dstname, "w");
    if (!fpsrc || !fpdst) { printf("fopen error\n"); return -1; }
    int ret;
    while ((ret = fgetc(fpsrc)) != EOF) {
        fputc(ret, fpdst);
    }
    fclose(fpsrc);
    fclose(fpdst);
}
```

### 4.4 fgets / fputs

```c
char *fgets(char *s, int size, FILE *stream);  // 读一行，最多size-1字节，保留\n
int fputs(const char *s, FILE *stream);         // 写一行，不写\0
```

fgets vs gets：
- fgets安全：限长size-1，最后一个字节存\0；gets不限长，危险
- fgets保留\n；gets丢弃\n
- fgets可读任意流；gets只能读终端
- 只能操作ASCII文本文件，不能操作二进制文件

### 4.5 行拷贝示例

```c
int copy_file(const char *srcname, const char *dstname) {
    FILE *fpsrc = fopen(srcname, "r");
    FILE *fpdst = fopen(dstname, "w");
    if (!fpsrc || !fpdst) { printf("fopen error\n"); return -1; }
    char buff[1024] = {0};
    while (fgets(buff, sizeof(buff), fpsrc) != NULL) {
        fputs(buff, fpdst);
    }
    fclose(fpsrc);
    fclose(fpdst);
}
```

### 4.6 fread / fwrite

```c
size_t fwrite(const void *ptr, size_t size, size_t nmemb, FILE *stream);
size_t fread(void *ptr, size_t size, size_t nmemb, FILE *stream);
```

- ptr：数据首地址；size：单个元素字节；nmemb：元素个数
- 返回实际操作的元素个数
- 既能读写文本，也能读写二进制文件
- 多用于固定大小数据，如结构体读写

### 4.7 文件流定位

```c
int fseek(FILE *stream, long offset, int whence);
long ftell(FILE *stream);
void rewind(FILE *stream);
```

whence取值：

| 常量 | 含义 | offset |
|------|------|--------|
| SEEK_SET | 文件开头 | >0 |
| SEEK_CUR | 当前位置 | 正负均可 |
| SEEK_END | 文件末尾 | 正负均可 |

求文件大小经典套路：

```c
fseek(fp, 0, SEEK_END);
long len = ftell(fp);
rewind(fp);
```

### 4.8 格式化输出家族

```c
printf(format, ...);           // → stdout
fprintf(stream, format, ...);  // → 指定文件流
sprintf(str, format, ...);     // → 内存字符串
```

## 5. 文件IO函数

### 5.1 open / close

```c
int open(const char *pathname, int flags);
int open(const char *pathname, int flags, mode_t mode);  // 有O_CREAT时必须给mode
int close(int fd);
```

flags组合：

| 标准IO mode | 等价 flags |
|-------------|-----------|
| "r" | O_RDONLY |
| "r+" | O_RDWR |
| "w" | O_WRONLY | O_CREAT | O_TRUNC |
| "w+" | O_RDWR | O_CREAT | O_TRUNC |
| "a" | O_WRONLY | O_CREAT | O_APPEND |
| "a+" | O_RDWR | O_CREAT | O_APPEND |

mode：权限位，如 0664；实际权限 = mode & ~umask

文件描述符fd：0~1023，分配原则——最小未被使用。
0/1/2 已被占用（stdin/stdout/stderr）。

不关闭文件会造成fd泄露（总共才1024个）。

### 5.2 read / write

```c
ssize_t write(int fd, const void *buf, size_t count);
ssize_t read(int fd, void *buf, size_t count);
```

- 成功返回实际操作字节数；read到末尾返回0；出错返回-1
- write不一定写入全部count字节，需要循环写

### 5.3 文件IO拷贝示例

```c
int copy_file(const char *srcname, const char *dstname) {
    int fdsrc = open(srcname, O_RDONLY);
    int fddst = open(dstname, O_WRONLY | O_CREAT | O_TRUNC, 0664);
    if (fdsrc < 0 || fddst < 0) { printf("open error\n"); return -1; }
    char buff[1024] = {0};
    while (1) {
        ssize_t size = read(fdsrc, buff, sizeof(buff));
        if (size <= 0) break;  // 0=末尾, <0=出错
        write(fddst, buff, size);
    }
    close(fdsrc);
    close(fddst);
}
```

### 5.4 lseek

```c
off_t lseek(int fd, off_t offset, int whence);
```

和fseek用法相同（SEEK_SET/SEEK_CUR/SEEK_END），返回偏移量。

求文件大小：`off_t len = lseek(fd, 0, SEEK_END);`

## 6. 标准IO vs 文件IO 核心区别

| | 文件IO | 标准IO |
|--|--------|--------|
| 面向 | fd（整数描述符） | FILE*（流指针） |
| 来源 | 系统调用 | C库函数 |
| 缓冲 | 无 | 有 |
| 可移植 | 弱 | 强 |
| 适合 | 硬件/管道/套接字 | 普通文本文件 |

## 7. 缓冲区机制

高速设备匹配低速设备的中间层：

### 行缓冲（1KB）——stdout / 终端

刷新条件：遇\n、缓冲区满、程序结束、fflush(stdout)

### 全缓冲（4KB）——普通文件

刷新条件：缓冲区满、程序结束、fflush(fp)、文件关闭

### 无缓冲（0KB）——stderr

直接刷新，不缓存

实战：`fprintf(stderr, "fopen error %s", filename);` 出错信息立刻输出

## 8. 出错处理

### errno

全局错误码变量，函数出错时自动写入。

```c
char *strerror(int errnum);        // 错误码 → 描述字符串
void perror(const char *s);        // 打印 "s: 错误原因"
void error(int status, int errnum, const char *format, ...);
```

error() 参数：status（0成功/1失败）、errnum（errno）、format（自定义信息）

内置宏：`__FILE__`、`__func__`、`__LINE__`、`__DATE__`

示例：`error(1, errno, "%s:%s:%d: open error", __FILE__, __func__, __LINE__);`

## 9. 目录操作

```c
DIR *opendir(const char *name);
struct dirent *readdir(DIR *dirp);
int closedir(DIR *dirp);
```

dirent结构体关键字段：`d_ino`（inode号）、`d_type`（文件类型）、`d_name[256]`（文件名）

```c
int mkdir(const char *pathname, mode_t mode);      // 创建目录，权限如0777
char *getcwd(char *buf, size_t size);               // 获取当前路径
int chdir(const char *path);                        // 切换工作路径
```

shell命令对照：`chmod 0777 1.txt`（改权限）、`pwd`（看路径）

## 10. 时间函数

```c
time_t time(time_t *tloc);              // 1970-1-1以来的秒数
char *ctime(const time_t *timep);       // 秒数 → 字符串时间
struct tm *localtime(const time_t *timep); // 秒数 → 结构体时间
```

tm结构体：tm_sec/tm_min/tm_hour/tm_mday/tm_mon(0-11)/tm_year(从1900起)/tm_wday/tm_yday/tm_isdst

注意：tm_mon从0开始，tm_year要加1900

## 速查表

| 操作 | 标准IO | 文件IO |
|------|--------|--------|
| 打开 | fopen → FILE* | open → fd |
| 关闭 | fclose | close |
| 读字符 | fgetc | — |
| 写字符 | fputc | — |
| 读行 | fgets | — |
| 写行 | fputs | — |
| 块读 | fread | read |
| 块写 | fwrite | write |
| 定位 | fseek/ftell/rewind | lseek |
| 格式化 | fprintf/sprintf | — |

| 缓冲类型 | 大小 | 场景 | 刷新触发 |
|----------|------|------|----------|
| 行缓冲 | 1KB | stdout终端 | \n / 满 / 结束 / fflush |
| 全缓冲 | 4KB | 普通文件 | 满 / 结束 / fflush / 关闭 |
| 无缓冲 | 0KB | stderr | 直接输出 |

### 易错提醒

1. fopen失败返回NULL，必须检查，否则后续读写直接崩
2. fgets只读size-1字节，末尾自动补\0；gets无限长，已废弃
3. fread返回0不代表出错，也可能是文件末尾——需要结合feof/ferror判断
4. O_CREAT必须配合mode参数，否则权限随机
5. write不一定写完count字节，需要循环确认
6. 行缓冲在终端下遇\n才刷新，如果printf没有\n，内容可能看不到
7. tm_mon从0开始（1月=0），tm_year要+1900才是实际年份
8. 文件描述符总共1024个，不close会泄露
