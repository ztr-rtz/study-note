# 嵌入式平台数据库

> 来源：Linux应用软件编程 — 四、嵌入式平台数据库  
> 整理日期：2026-07-15

## 1. 为什么用数据库而不是普通文件

普通文件做增删改查麻烦，效率低。数据库专门管数据，查起来快，也方便维护。

## 2. 数据库分类

### 关系型数据库
数据组织成二维表格。
- 大型：Oracle、DB2
- 中型：MySQL、SQL Server
- 小型：SQLite

### 非关系型数据库
键值对存储，结构不固定。
- Redis
- MongoDB

### 嵌入式数据库
典型代表是 SQLite3，特点很鲜明：
- 开源免费，用 C 写的
- 代码量小，约 1 万行，整体 10M 以内
- 文件型数据库，一个 .db 文件就能搬来搬去
- 容量最大能到 2T

## 3. SQLite3 安装

### 在线装
```bash
sudo apt-get install sqlite3
sudo apt-get install libsqlite3-dev
```

### 源码装
```bash
tar -zxvf sqlite-autoconf-3080403.tar.gz
cd sqlite-autoconf-3080403
./configure
make
sudo make install
```

## 4. sqlite3 常用命令

进入数据库后，点号开头的是 sqlite3 自己的命令：

| 命令 | 作用 |
|------|------|
| .help | 查看所有支持的命令 |
| .quit | 退出 sqlite3 |
| .tables | 查看当前数据库里的表 |
| .schema 表名 | 查看建表语句 |
| .headers on/off | 开关列名显示 |
| .mode column | 左对齐列宽显示 |
| .width 列1宽 列2宽 ... | 设置每列显示宽度 |

## 5. SQL 语句速查

SQLite 支持标准 SQL，数据类型比较简单：
- `INTEGER`：整数
- `REAL`：浮点
- `TEXT`：字符串
- `NULL`：空值

### 建表
```sql
create table 表名(列1 类型, 列2 类型, ...);
-- 如果表不存在才创建
create table if not exists 表名(...);
-- 主键自动增长
create table 表名(id INTEGER PRIMARY KEY AUTOINCREMENT, ...);
```

### 插入
```sql
insert into 表名 values(值1, 值2, ...);
```

### 查询
```sql
select * from 表名;
select 列名 from 表名;
select 列1, 列2 from 表名;
```

### 条件查询
```sql
select 列名 from 表名 where 列 运算符 值;
-- 运算符：> >= = != < <= and or
```

### 模糊查询
```sql
select * from 表名 where 列名 like '%冬梅';  -- 以冬梅结尾
select * from 表名 where 列名 like '马%';    -- 以马开头
select * from 表名 where 列名 like '马__';  -- 马 + 两个字符
-- % 代表任意个字符，_ 代表一个字符
```

### 排序
```sql
select * from 表名 order by 列名 ASC;   -- 升序
select * from 表名 order by 列名 DESC;  -- 降序
```

### 删除
```sql
delete from 表名 where 列名 = 值;
```

### 修改
```sql
update 表名 set 列名 = 新值 where 列名 = 某值;
```

### 时间函数
```sql
select datetime('now', '+8 hours');  -- 东八区当前时间
insert into 表名 values(1, 'zhangsan', 99, datetime('now', '+8 hours'));
```

### 删表
```sql
drop table 表名;
```

## 7. C/C++ 操作 SQLite3

### 编程框架
打开数据库 → 读写数据库（增删改查） → 关闭数据库

编译时要链接 sqlite3 库：
```bash
gcc xxx.c -lsqlite3
```

### 核心 API

#### 1. sqlite3_open
```c
int sqlite3_open(const char *filename, sqlite3 **ppDb);
```
- 打开数据库文件，获得数据库句柄
- 文件存在就打开，不存在就创建
- 成功返回 `SQLITE_OK`

#### 2. sqlite3_errmsg
```c
const char *sqlite3_errmsg(sqlite3 *);
```
- 拿到出错原因字符串

#### 3. sqlite3_close
```c
int sqlite3_close(sqlite3 *);
```
- 关闭数据库句柄，释放资源

#### 4. sqlite3_exec
```c
int sqlite3_exec(
    sqlite3 *pDb,
    const char *sql,
    int (*callback)(void*, int, char**, char**),
    void *arg,
    char **errmsg
);
```
- 执行一条 SQL 语句
- `callback` 只在 `select` 时用到，每查到一行就调用一次
- `arg` 传给回调函数的用户参数
- `errmsg` 存放错误信息，用完要用 `sqlite3_free` 释放
- 成功返回 `SQLITE_OK`

回调函数格式：
```c
int fun(void *para, int n_column, char **column_value, char **column_name);
```
- `para`：`sqlite3_exec` 第四个参数传进来的
- `n_column`：这一行有几列
- `column_value`：每列的值（字符串形式）
- `column_name`：每列的字段名

注意：
- 查出来数据全是字符串
- 回调函数必须有返回值，否则 `sqlite3_exec` 会失败

#### 5. sqlite3_free
```c
void sqlite3_free(void *);
```
- 释放 `sqlite3_exec` 返回的错误信息空间

## 8. 易错提醒

1. `sqlite3_exec` 的 SQL 字符串末尾不需要加分号，函数内部会处理。
2. `select` 查询结果的数字在回调里也是字符串，需要自己转。
3. 回调函数必须返回 0，否则整个 `sqlite3_exec` 调用失败。
4. 出错信息 `errmsg` 不是系统自动回收的，必须手动 `sqlite3_free`。
5. 编译时别忘记 `-lsqlite3`。
6. 建表时 `AUTOINCREMENT` 只能和 `INTEGER PRIMARY KEY` 一起用。
7. `like` 模糊查询里 `%` 匹配任意多个字符，`_` 只匹配一个字符。
