# C语言流程控制

> 从第三章笔记整理 —— 分支、循环、跳转，写逻辑全靠这几板斧。

---

## 一、三种基本结构

C 语言代码执行就三种路子：

1. **顺序执行** —— 从 main 开始，一条一条往下走，最简单。
2. **分支选择** —— 根据条件决定走哪条路，非此即彼。
3. **循环重复** —— 一段代码反复跑，直到条件不满足。

---

## 二、关系运算符（比较运算符）

用来比大小的，操作数得是数值型（int、float、double 这些）。

| 运算符 | 含义 | 注意 |
|--------|------|------|
| `>` | 大于 | |
| `>=` | 大于等于 | |
| `<` | 小于 | |
| `<=` | 小于等于 | |
| `==` | 等于 | 别和 `=`（赋值）搞混 |
| `!=` | 不等于 | |

比较的结果是 int 类型：真 = 1，假 = 0。但 C 语言里判断真假不只看 0 和 1——**非 0 即为真**，记住这个就行。

---

## 三、逻辑运算符

操作对象是逻辑值（真假），结果也是 int 类型（1 真 / 0 假）。

| 运算符 | 含义 | 说明 |
|--------|------|------|
| `&&` | 逻辑与（AND） | 两边都为真才真 |
| `||` | 逻辑或（OR） | 一边为真就真 |
| `!` | 逻辑非（NOT） | 取反，真变假、假变真 |

### 短路效应（截断效应）

这是 `&&` 和 `||` 最容易踩坑的地方：

- `0 && b++` —— 左边已经是 0 了，右边 **根本不执行**，b 的值不会变。
- `1 || ++b` —— 左边已经是 1 了，右边的 `++b` **直接跳过**。

结论：**不要往逻辑表达式的右侧塞带副作用的操作（++、--、函数调用等）**，你以为会执行，实际可能没跑。

---

## 四、三目运算符 `?:`

```c
表达式1 ? 表达式2 : 表达式3
```

- 表达式1 为真 → 整个式子的值就是表达式2 的值
- 表达式1 为假 → 整个式子的值就是表达式3 的值
- 结合方向：**从右向左**

典型用法：

```c
max = num1 > num2 ? num1 : num2;  // 找最大值，一行搞定
```

## 五、分支结构

### 5.1 单分支 if

```c
if (表达式) {
    // 表达式为真才执行
}
```

注意：`if (表达式);` 后面加分号相当于空语句体，语法不报错但逻辑全乱，别踩这个坑。

### 5.2 双分支 if-else

```c
if (表达式) {
    // 真走这
} else {
    // 假走这
}
```

else 不能单独出现，它跟最近的 if 配对。

### 5.3 阶梯分支 else-if

多条件判断的经典写法：

```c
if (表达式1) {
    // ...
} else if (表达式2) {
    // ...
} else if (表达式3) {
    // ...
} else {
    // 以上全不满足
}
```

### 5.4 if 嵌套

```c
if (条件1) {
    if (条件2) {
        if (条件3) {
            // 等价于 if (条件1 && 条件2 && 条件3)
        }
    }
}
```

嵌套别超过 3-5 层，多了自己都看不懂。

### 5.5 switch-case

多分支等值匹配场景用 switch 比 if-else 更清晰：

```c
switch (整型表达式) {
    case 整型常量1:
        语句块1;
        break;
    case 整型常量2:
        语句块2;
        break;
    default:
        默认语句块;
}
```

三条硬规则：

1. **switch 后面必须是整型表达式**（int、char、short 都行，float 不行）
2. **case 后面必须是整型常量**，不能是变量
3. 匹配到 case 之后，一直往下执行所有 case，**直到遇到 break 才跳出去**——这跟大多数语言不一样，小心穿透。

没有匹配的 case 就走 default，default 可写可不写，一般放最后。

## 六、循环结构

写循环前先想清楚三个要素：

1. 循环变量在哪初始化
2. 循环继续的条件是什么
3. 循环变量怎么变（++ / --）

### 6.1 while 循环

```c
while (表达式) {
    循环体;
}
```

执行逻辑：先判断 → 为真执行循环体 → 再判断 → 循环……

- 适合**不知道要跑多少次**的场景
- `while(1) {}` = 死循环
- 别在 while 后面加分号

### 6.2 for 循环

```c
for (表达式1; 表达式2; 表达式3) {
    循环体;
}
```

| 表达式 | 作用 | 执行次数 |
|--------|------|----------|
| 表达式1 | 初始化循环变量 | 只跑 1 次 |
| 表达式2 | 循环条件判断 | 每次循环前判断 |
| 表达式3 | 控制变量变化（++/--） | 每次循环体执行完后跑 |

- 适合**知道要跑多少次**的场景
- 死循环写法：`for(;;)`，两个分号不能省

### 6.3 do-while 循环

```c
do {
    循环体;
} while (表达式);
```

和 while 的唯一区别：**循环体至少执行一次**，先跑了再判断。

`while()` 后面的分号别忘，否则编译报错。

### 6.4 双层循环

```c
for (j = 0; j < 3; j++) {     // 外层
    for (i = 0; i < 5; i++) { // 内层
        // ...
    }
}
```

外层走 1 次，内层跑完一整轮。上面这个例子：外层 3 次 × 内层 5 次 = 总共 15 次。

**经典练习 —— 九九乘法表：**

```c
for (j = 1; j <= 9; j++) {
    for (i = 1; i <= j; i++) {
        printf("%d*%d=%d ", i, j, i * j);
    }
    puts("");
}
```

## 七、循环辅助控制

### 7.1 continue —— 跳过本次

```c
for (i = 0; i < 10; i++) {
    if (i == 3 || i == 5 || i == 7) {
        continue;  // 遇到 3、5、7 直接跳过，不打印
    }
    printf("i = %d\n", i);
}
```

只跳过当次循环，循环本身继续。

### 7.2 break —— 终止循环

```c
for (i = 1; i < 10; i++) {
    if (i == 3 || i == 5 || i == 7) {
        break;  // 遇到 3 就直接结束了
    }
    printf("i = %d\n", i);
}
printf("end...\n");
```

直接跳出**当前**循环（如果是嵌套循环，只跳最内层）。break 也能用于跳出 switch。

## 八、goto（了解即可）

```c
loop:
    sum += i;
    i++;
    if (i <= 10) goto loop;
```

等价于汇编的 jmp 指令，只能在**同一个函数内部**跳转。

**应用层编程禁止使用 goto**，打乱代码流程，难读难调试。Linux 内核里错误处理场景用得多，但这跟日常开发没啥关系。

## 速查表

| 结构 | 使用场景 | 关键陷阱 |
|------|----------|----------|
| if-else | 条件判断 | else 跟最近的 if 配对 |
| else-if | 多条件阶梯判断 | 条件顺序要合理 |
| switch | 多值等值匹配 | 别忘 break，只能整型 |
| while | 未知循环次数 | 别在括号后加分号 |
| for | 已知循环次数 | `for(;;)` 死循环 |
| do-while | 至少执行一次 | while 后分号必写 |
| continue | 跳过当次循环 | 只跳一次，不终止 |
| break | 提前终止循环/switch | 只跳最内层 |
| goto | 内核错误处理 | **应用层别用** |
