# C语言预处理命令

预处理在编译之前执行，处理所有 `#` 开头的指令。gcc 编译流程：预处理 → 汇编 → 编译 → 连接，预处理指令必须在第一步全部完成。

---

## 1. `#include` — 文件包含

`#include` 的本质是文本复制：把被包含文件的内容直接粘贴到当前位置。

头文件里通常放全局变量声明、函数声明、自定义类型声明、别名、宏名。

两种写法：

| 写法 | 搜索路径 |
|------|----------|
| `#include <stdio.h>` | 系统头文件目录（`/usr/include`） |
| `#include "myhead.h"` | 先找当前目录（a.out 所在目录），再找系统目录 |

不要包含 `.c` 文件。多文件编程用 `.c` + `.h` 模板组织。

---

## 2. `#define` — 宏定义

宏的核心：**纯文本替换**，不做任何计算。替换发生在预处理阶段，编译器拿到的是替换后的结果。

### 2.1 不带参宏（对象式宏）

```c
#define PI 3.14
#define M 50
#define N ((M)+(M))   // 宏值是表达式时，加圆括号
```

使用：

```c
double area = r * r * PI;   // PI → 3.14
int num = N * N;             // ((M)+(M)) * ((M)+(M)) = 10000
```

**圆括号是刚需。** 如果写成 `#define N M+M`，那 `N*N` 会展开成 `M+M*M+M` = 2600，不是期望的 10000。

### 2.2 带参宏（函数式宏）

```c
#define MAX(x,y)  ((x)>(y) ? (x) : (y))
#define ADD(a,b)  ((a)+(b))
#define LENTH(r)  (2*(PI)*(r))
```

参数和整体都要加括号，原因和不带参宏一样——防止展开后运算优先级出错。

例：`ADD(3>4, 5)` 展开为 `((3>4)+(5))` = `0+5` = 5。如果没括号，`3>4+5` 结果完全不同。

多行带参宏用 `do{...}while(0)` 包裹：

```c
#define PRINT(a,b,c) do{\
    printf("a is %d\n",a);\
    printf("b is %d\n",b);\
    printf("c is %d\n",c);\
}while(0)
```

### 2.3 带参宏 vs 函数

| | 带参宏 | 函数 |
|---|---|---|
| 处理时机 | 预处理阶段，文本替换 | 运行阶段，调用执行 |
| 参数类型 | 无类型，纯替换 | 有类型，值/地址传递 |
| 内存开销 | 每次调用都展开一份代码，调用多则代码膨胀 | 运行时只有一份代码 |
| 执行速度 | 无跳转、无传参，直接执行 | 有传参和跳转开销 |
| 适用场景 | 功能简单、一两行表达式 | 功能复杂、多行逻辑 |

一句话总结：宏是"写时替换"，函数是"用时调用"。

## 3. 条件编译（宏开关）

条件编译让同一份源码在不同条件下编译出不同版本，调试、选配功能、防止重复包含都靠它。

### 3.1 `#if` / `#elif` / `#else` / `#endif`

```c
#define DEBUG 1

#if DEBUG
    // 调试代码，正式发布时把 DEBUG 改成 0 就不编译了
#elif VERSION > 3
    // 另一个分支
#else
    // 默认分支
#endif
```

`#if` 判断条件真假，决定 `#if` 和 `#endif` 之间的代码是否进入编译。

### 3.2 `#ifdef` / `#ifndef` — 宏是否已定义

```c
#ifndef __ADD_H_
#define __ADD_H_
    // 头文件内容
#endif
```

这段是头文件的标准防护写法：

- `#ifndef __ADD_H_`：如果 `__ADD_H_` 没被定义过，进入内部
- `#define __ADD_H_`：定义它
- 下次再 `#include` 同一个头文件时，`__ADD_H_` 已经存在，整个块被跳过

作用：防止头文件被多次包含导致重复声明。

`#ifdef` 反过来用——宏已定义才编译：

```c
#ifdef __ADD_H_
    // 选配功能代码
#endif
```

适合做可选功能的开关。

## 速查表

| 指令 | 作用 | 关键要点 |
|------|------|----------|
| `#include <...>` | 包含系统头文件 | 只搜系统目录 |
| `#include "..."` | 包含自定义头文件 | 先搜当前目录，再搜系统目录 |
| `#define NAME val` | 不带参宏 | 宏值是表达式时加圆括号 |
| `#define NAME(a,b) expr` | 带参宏 | 参数和整体都加括号 |
| `#if cond` | 条件编译 | 判断条件真假决定编译 |
| `#ifdef NAME` | 宏已定义才编译 | 选配功能开关 |
| `#ifndef NAME` | 宏未定义才编译 | 头文件防重复包含的标准写法 |
| `#elif` / `#else` | 条件分支 | 配合 `#if` 使用 |
| `#endif` | 结束条件编译块 | 必须配对 |

## 易错提醒

1. **宏不加括号**是头号坑。`#define SQUARE(x) x*x`，传 `SQUARE(1+2)` 展开成 `1+2*1+2` = 5，不是 9。
2. **不要 `#include .c 文件**，会导致重复定义。
3. **`#ifndef` 防护**每个自定义头文件都要加，不是可选的。
4. **带参宏没有类型检查**，传什么它都替换，调试时不如函数直观。
5. **`#if 0` ... `#endif`** 可以临时屏蔽一段代码，比逐行注释方便。
