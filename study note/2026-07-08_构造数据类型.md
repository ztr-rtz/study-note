# 构造数据类型（自定义数据类型）

C 里除了 int、char 这些内置类型，还允许你自己组合出新的类型。构造数据类型分三种：结构体、共用体、枚举。

---

## 一、结构体（struct）

### 干嘛用的

把一堆不同类型的变量打包在一起，描述一个"东西"。比如描述一个人，需要姓名、年龄、身高、电话，甚至还嵌套一个出生日期。

### 怎么声明

```c
struct 结构体名 {
    成员1;
    成员2;
    ...
};  // 分号别漏
```

声明的时候不开辟内存，只是告诉编译器"有这么个结构"。

```c
struct Date {
    int year;
    int mon;
    int day;
};

struct Person {
    char name[50];
    int age;
    int height;
    char phone[20];
    struct Date date;   // 嵌套另一个结构体
};
```

### 定义变量 & 访问成员

```c
struct Person per;            // 开辟内存
struct Person *p = &per;

// 变量用 .   指针用 ->
per.name;      // 类型就是 name 的类型（char[]）
p->age;        // 类型就是 age 的类型（int）
```

. 和 -> 都是 1 级优先级，从左往右结合。记住一条规则就够了：左边是变量用点，是指针用箭头。

### 结构体的大小 ≠ 口算之和

```c
struct Person {
    char name[50];   // 50
    int age;         // 4
    float height;    // 4
    char phone[20];  // 20
};
// 口算：50+4+4+20 = 78
// 实际：sizeof = 80
```

**字节对齐**：编译器为了 CPU 读写效率，会在成员之间塞填充字节。

两条规则：
- 每个成员的起始地址必须能被自身类型大小整除（地址 % sizeof(类型) == 0）
- 整个结构体的大小必须是最大基本类型成员的整数倍（4 或 8 的倍数）

所以上面那个结构体，phone[20] 之后补了 2 字节凑成 80。

### 结构体传参

**值传递：**

```c
void fun(struct Person per) { ... }
```

很少这么写。形参是整个结构体的拷贝，结构体大的时候又耗内存又慢。

**地址传递（推荐）：**

```c
void fill_per(struct Person *p) {    // 要改数据
    scanf("%s", p->name);
    scanf("%d", &p->id);
}

void show_per(const struct Person *p) {  // 只读
    printf("name:%s id:%d\n", p->name, p->id);
}
```

指针固定 8 字节，不管结构体多大。只读就加 const，要改就去掉。

### 结构体清零

两种方式：

```c
struct Person per = {0};                // 初始化全零
memset(&per, 0, sizeof(per));          // 手动清
```

### 结构体数组

```c
// 全清零
struct Person arr[5] = {0};

// 逐个初始化
struct Person arr2[3] = {
    {"zhangsan", 10, {2026, 5, 10}},
    {"lisi",     11, {2025, 3, 12}},
    {"wangmazi", 12, {2024, 1, 22}},
};

// GCC 扩展：指定下标初始化（不推荐的写法，移植性差）
struct Person arr3[3] = {
    [1] = {"zhangsan", 20, {2021, 10, 1}},
};
```

遍历跟普通数组一样，下标访问即可。

### typedef 简化写法

每次写 `struct Person` 有点啰嗦，用 typedef 给个短名：

```c
typedef struct Person1 {
    char name[50];
    int id;
} PER;

// 之后直接用 PER 代替 struct Person1
PER p1 = {"zhangsan", 20};
struct Person1 p2 = {"lisi", 10};  // 老写法也兼容
```

### 简单对比：声明 vs 定义

| 操作 | 是否开辟内存 |
|------|-------------|
| `struct Person { ... };` （声明类型） | 否 |
| `struct Person per;` （定义变量） | 是 |

## 二、共用体（union）

定义、初始化、访问语法和结构体一模一样。区别只有一点：**怎么分配内存**。

- 结构体：每个成员各自占一块空间，互不冲突
- 共用体：所有成员共用同一块空间，大小等于最大成员的大小

### 什么时候用

- 同一时刻只需要用其中一个成员，想省内存
- 函数参数有多种可能的类型（多选一）
- 判断 CPU 大小端

### 大小端判断

```c
union End {
    int ia;
    char cc;
};

union End end = {0};
end.ia = 1;
if (end.cc == 1)
    printf("小端\n");   // 低地址存低位字节
else
    printf("大端\n");
```

原理：int 占 4 字节，char 只读第一个字节。如果 ia=1，小端机器上低字节是 01，cc 读到 1；大端机器上低字节是 00。

## 三、枚举（enum）

把变量可能取的值一个一个列出来。本质上就是给整数常量起个名字，编译器当 int 处理。

```c
enum WEEK {
    MON, TUE, WED, THU, FRI, SAT, SUN
};
// MON=0, TUE=1, ..., SUN=6
```

默认从 0 开始，依次 +1。也可以手动指定：

```c
enum WEEK1 {
    MON1 = 3, TUE1, WED1, THU1, FRI1, SAT1, SUN1
};
// 3, 4, 5, 6, 7, 8, 9
```

使用：

```c
enum WEEK today = THU;   // today = 3

switch (today) {
case MON:  printf("上学\n"); break;
case FRI:  printf("出去玩\n"); break;
case SUN:  printf("睡觉\n"); break;
default:   printf("发呆\n");
}
```

虽然可以 `today = 1000`，但没人这么写——枚举的意义就是让代码可读，硬塞魔法数字就失去意义了。

## 速查表

| 类型 | 关键字 | 内存分配 | 成员关系 | 典型用途 |
|------|--------|----------|----------|----------|
| 结构体 | struct | 各成员独立空间，有字节对齐 | 并存 | 描述复合数据 |
| 共用体 | union | 等于最大成员大小，共享 | 互斥 | 省内存、大小端判断 |
| 枚举 | enum | 等价 int（4 字节） | 一个值 | 限制取值范围、提高可读性 |

### 易错点

- 结构体声明末尾的分号别忘了
- 地址传递才是结构体传参的正确姿势，值传递浪费内存
- 字节对齐会导致 sizeof 不等于各成员之和
- 共用体同一时刻只用一个成员，后面的赋值会覆盖前面的
- 枚举变量在 C 里就是 int，可以赋任意整数（但别这么干）
