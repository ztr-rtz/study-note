## 1. Ubuntu 终端操作

系统版本：Ubuntu 18.04。

启动终端：`Ctrl + Alt + T`，或者从快速启动栏点进去。

几个常用快捷键：

| 快捷键 | 作用 |
|---|---|
| `Ctrl + Shift + +` | 终端字体放大 |
| `Ctrl + Shift + -` | 终端字体缩小 |
| `Ctrl + Shift + C` | 复制文本 |
| `Ctrl + Shift + V` | 粘贴文本 |
| `Tab` | 命令 / 文件名自动补齐 |
| 上下方向键 | 浏览历史命令 |

查看操作系统版本：

```bash
cat /etc/issue
```

## 2. Shell 命令提示符

```text
linux@ubuntu:~$
```

- `linux` —— 当前用户名
- `ubuntu` —— 主机名
- `~` —— 当前目录，`~` 代表家目录（会随 cd 切换变化）
- `$` —— 普通用户；如果是 `#`，说明当前是 root（系统管理员）

命令通用格式：

```text
command [-options] arg1 arg2 ...
```

## 3. 文件与目录操作

### ls —— 查看目录内容

```bash
ls          # 列出当前目录下可见文件/目录
ls -a       # 包含隐藏文件（. 开头）
ls -l       # 长格式，等价于 ll
```

`ls -l` 输出解读：

```text
drwxrwxr-x  6 linux linux  4096  Feb  6 14:03  20260113
-rw-rw-r--  1 linux linux   278  Dec  4 14:14  2.c
```

第一列拆开看（以 `drwxrwxr-x` 为例）：

| 位 | 含义 |
|---|---|
| `d` | 类型：`d` 目录，`-` 普通文件 |
| `rwx` | 属主权限（读/写/执行） |
| `rwx` | 属组权限 |
| `r-x` | 其他人权限 |

其余字段：硬链接数 → 属主 → 属组 → 文件大小(字节) → 最后修改时间 → 文件名。

硬链接数：对文件来说，是有几个文件名关联到同一份内容；对目录来说，是该目录下有几个子目录。

### mkdir / rmdir —— 创建 / 删除目录

```bash
mkdir 20260422      # 创建目录
rmdir 20260422      # 删除空目录，目录非空会失败
```

### cd —— 切换目录

```bash
cd 20260422     # 进入子目录
cd ..           # 返回上一层
cd /            # 跳到根目录（最高层）
cd              # 回到家目录（等价 cd ~）
```

一直 `cd ..` 最终会到 `/`（根目录）。根目录下常见子目录：

```text
bin  boot  dev  etc  home  lib  lib32  lib64
media  mnt  opt  proc  root  run  sbin  srv
sys  tmp  usr  var
```

相对路径 vs 绝对路径：

- 相对路径：以当前目录为基准，比如 `test1/aaa`
- 绝对路径：从 `/` 开始写全，比如 `/home/linux/20260422/test2/bbb`

### touch —— 创建空文件

```bash
touch 123       # 新建空文件；如果 123 已存在，只更新修改时间戳
```

### rm —— 删除文件 / 目录

```bash
rm 123              # 删除文件
rm -fr 目录名        # 递归强制删除目录（-r 递归，-f 不询问）
```

### pwd —— 显示当前路径

```bash
pwd
# 输出：/home/linux/20260422/aaa/bbb
```

### cp —— 复制

复制单个文件：

```bash
cp 源文件 目标文件
# 目标文件不存在 → 新建；存在 → 覆盖
```

复制整个目录（带 `-av`）：

```bash
cp ../14klist/ . -av
```

### mv —— 移动 / 重命名

```bash
mv 1.c 14klist/     # 把 1.c 移到 14klist 目录下
mv app bpp          # 把 app 重命名为 bpp
```

### cat —— 查看文本文件内容

```bash
cat 1.txt
```

## 4. Vim 基础操作

命令行文本编辑器，主要用来写 C 代码。

基本流程：

```text
1. vim 1.txt          → 打开文件
2. 按 i               → 进入编辑模式（左下角显示 INSERT）
3. 编辑内容
4. 按 Esc，输入 :wq   → 保存退出
```

命令模式常用操作（左下角显示 NORMAL）：

| 命令 | 说明 |
|---|---|
| `yy` | 复制当前行 |
| `dd` | 剪切/删除当前行 |
| `p` | 粘贴 |
| `u` | 撤销 |
| `3yy` | 从当前行向下复制 3 行 |
| `5dd` | 从当前行向下删除 5 行 |
| `:3,10y` | 复制第 3~10 行 |
| `:2,5d` | 删除第 2~5 行 |
| `:wq` | 保存退出 |
| `:q!` | 不保存退出 |
| `/关键词` | 搜索，`n` 下一个，`N` 上一个 |
| `gg` | 跳到文件开头 |
| `GG` / `G` | 跳到文件末尾 |

全选 + 自动格式化：`gg` → `v` → `GG` → `=`

分屏：

```bash
:vs 1.c     # 竖分屏
:sp 1.c     # 横分屏
:wqa        # 全部保存退出
```

## 5. GCC 编译

把 C 源文件编译成可执行文件。

```bash
gcc 1.c         # 默认生成 a.out
gcc 1.c -o app  # 指定输出文件名为 app
./app           # 执行
```

### 编译四阶段

```text
1. 预处理   gcc 1.c -E -o 1.i    处理 # 开头的指令（#include, #define）
2. 编译     gcc 1.c -S -o 1.s    C 代码 → 汇编代码
3. 汇编     gcc 1.c -c -o 1.o    汇编 → 目标文件（二进制，不可执行）
4. 链接     gcc 1.c -o app       关联外部库函数，生成可执行文件
```

各阶段说明：

- **预处理**：`#include` 把头文件内容粘贴过来；`#define PI 3.14` 全局替换
- **编译**：翻译成汇编语言
- **汇编**：转为 `.o` 目标文件。用 `od -t x1 1.o | less` 可以查看二进制内容
- **链接**：让 a.out 知道运行时去哪里找 `printf` 这些库函数的实现

### 编译报错两类

1. **警告（warning）**：能生成可执行文件，但代码有问题（如缺头文件）。建议修掉。
2. **错误（error）**：不会生成可执行文件，必须修。

## 6. 程序运行原理

1. 可执行文件 `a.out` 存放在外存（SSD / Flash）
2. `./a.out` 执行时，代码 + 数据被加载到内存
3. CPU 从内存读取代码和数据，执行运算

## 8. 进制转换

四种进制：

| 进制 | 规则 | 可用数码 |
|---|---|---|
| 十进制 | 逢十进一 | 0~9 |
| 二进制 | 逢二进一 | 0, 1 |
| 八进制 | 逢八进一 | 0~7 |
| 十六进制 | 逢十六进一 | 0~9, A~F |

### 十进制 → 其他进制

反复除以目标进制基数，记录余数，逆序排列。

例：十进制 23 转各进制：

```text
23 → 二进制：23÷2=11余1, 11÷2=5余1, 5÷2=2余1, 2÷2=1余0, 1÷2=0余1
          逆序：10111

23 → 八进制：23÷8=2余7, 2÷8=0余2 → 27

23 → 十六进制：23÷16=1余7, 1÷16=0余1 → 17
```

### 其他进制 → 十进制

每位乘以对应权值，然后全部相加。

例：`1000 1010`（二进制）→ 十进制

```text
1×2⁷ + 0×2⁶ + 0×2⁵ + 0×2⁴ + 1×2³ + 0×2² + 1×2¹ + 0×2⁰
= 128 + 8 + 2 = 138
```

### 快速互转

- 二进制 → 八进制：每 3 个二进制位 = 1 个八进制位
- 二进制 → 十六进制：每 4 个二进制位 = 1 个十六进制位
