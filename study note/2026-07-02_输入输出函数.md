# C语言输入输出函数

> 参考系：以内存为中心。数据进入内存叫"输入"，从内存出去叫"输出"。
> 本章函数均声明在 `<stdio.h>`。

---

## 1. 字符级 I/O

### putchar — 输出一个字符

```c
int putchar(int c);
```

- 把参数 `c` 的低 8 位作为字符输出到终端
- 参数可以是字符常量、字符变量、表达式

```c
putchar('h');           // 输出 h
putchar('h' + 1);       // 输出 i
char c = 'o';
putchar(c);             // 输出 o
```

### getchar — 从终端读一个字符

```c
int getchar(void);
```

- 阻塞等待用户输入，返回读取的字符（实际返回 int，失败返回 EOF）
- 调用时 `()` 不可省略

```c
char c = getchar();     // 等待用户按一个键
```

## 2. 格式化 I/O（核心）

### printf — 格式化输出

```c
int printf(const char *format, ...);
```

第一个参数是格式字符串，后面的参数按格式串中的 `%` 控制符依次填入。

#### 常用控制字符速查

| 控制符 | 含义 | 示例 |
|--------|------|------|
| `%d` | 有符号十进制整数 | `printf("%d", 42);` |
| `%u` | 无符号十进制整数 | `printf("%u", 42U);` |
| `%ld` / `%lu` | long 有符号 / 无符号 | `printf("%ld", 100L);` |
| `%c` | 按 ASCII 输出字符 | `printf("%c", 65);` → A |
| `%s` | 字符串 | `printf("%s", "hi");` |
| `%f` | float/double 小数形式 | `printf("%f", 3.14);` |
| `%lf` | double（scanf 中用，printf 中 `%f` 也兼容） | — |
| `%e` | 科学计数法 | `printf("%e", 3.14);` |
| `%o` / `%#o` | 八进制 / 带 `0` 前缀 | `printf("%#o", 8);` → 010 |
| `%x` / `%#x` | 十六进制 / 带 `0x` 前缀 | `printf("%#x", 255);` → 0xff |
| `%p` | 指针地址 | `printf("%p", &var);` |
| `%%` | 输出百分号本身 | `printf("50%%");` → 50% |

#### 宽度和精度控制

```c
printf("%5d",   42);   // "   42"   —— 右对齐，宽度 5
printf("%-5d",  42);   // "42   "   —— 左对齐
printf("%05d",  42);   // "00042"   —— 空位补 0
printf("%.2f",  3.1415);// "3.14"   —— 小数点后 2 位
```

#### 转义字符（本质是单个字符）

| 转义 | 含义 |
|------|------|
| `\n` | 换行 |
| `\t` | 制表符（跳到下一个制表位） |
| `\b` | 退格 |
| `\r` | 光标回行首 |
| `\"` | 双引号 |
| `\\` | 反斜杠本身 |

```c
printf("hello \"world\"\n");   // hello "world"
printf("1\t2\t3\n4\t5\t6\n");  // 两行三列表格效果
```

### scanf — 格式化输入

```c
int scanf(const char *format, ...);
```

#### 核心规则

1. **普通变量要加 `&`**，取地址传进去
2. **数组名不用加 `&`**（数组名本身就是地址）
3. 多个值之间用空格、Tab 或回车分隔
4. **格式串里不要加 `\n`**——会导致输入卡死
5. 格式匹配失败时，scanf 直接放弃后续读取

```c
int a, b;
scanf("%d%d", &a, &b);    // 输入 "10 20" 回车

float f;
scanf("%f", &f);          // float 用 %f

double d;
scanf("%lf", &d);         // double 必须用 %lf

char name[30];
scanf("%s", name);        // 数组不用 &
```

#### 踩坑：残留换行问题

连续用 scanf 后再读字符时，缓冲区残留的 `\n` 会被后面的 `%c` 直接吃掉。

```c
// 有问题版本
scanf("%d", &num);
scanf("%c", &c);          // 这里 c 会读到上一个回车

// 修复：加 getchar 吃掉残留换行
scanf("%d", &num);
getchar();                // 吃掉缓冲区里的 \n
scanf("%c", &c);          // 现在才真正等用户输入
```

## 3. 字符串 I/O

### puts — 输出字符串并自动换行

```c
int puts(const char *s);
```

比 printf 简单：只输出字符串，末尾自动加 `\n`。

```c
puts("hello");            // 输出 hello 并换行
char str[] = "world";
puts(str);                // 同上
```

### gets — 读取一行（不推荐，已废弃）

```c
char *gets(char *buf);
```

- 读到换行为止，换行符不存入 buf，自动补 `\0`
- **不检查缓冲区大小**，输入过长直接越界 → 栈崩溃

```c
char name[5];
gets(name);   // 输入超过 4 个字符就炸
```

现代替代：用 `fgets(buf, size, stdin)` 更安全。

---

## 4. 一张表总结

| 函数 | 方向 | 操作对象 | 注意 |
|------|------|----------|------|
| `putchar(c)` | 输出 | 单个字符 | 参数可以是表达式 |
| `getchar()` | 输入 | 单个字符 | 返回 int |
| `printf(...)` | 输出 | 格式化字符串 | 灵活强大，注意控制符匹配 |
| `scanf(...)` | 输入 | 格式化读取 | 变量前加 `&`，别加 `\n` |
| `puts(s)` | 输出 | 字符串 | 自动换行 |
| `gets(s)` | 输入 | 字符串 | ⚠️ 不检查越界，别用 |

---

## 要点回顾

- **输入 = 数据进内存，输出 = 数据出内存**，以此为基准理解方向
- **printf/scanf 是日常调试最常用的两个函数**，控制字符和变量必须一一对应
- **scanf 后接 `%c` 容易踩坑**，因为缓冲区残留的换行符会被当作字符读走，解法是中间加一个 `getchar()`
- **gets 不安全，已从 C11 标准移除**，改用 `fgets`
