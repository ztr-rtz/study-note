## 一、I2C 是什么？

**I2C**（Inter-Integrated Circuit，读作 "I 方 C"）是飞利浦（Philips，现 NXP）1982 年推出的**两线式同步串行半双工总线**。

只用两根信号线就能让多颗芯片互相通信：

- **SCL**（Serial Clock）：串行时钟线，由主机输出
- **SDA**（Serial Data）：串行数据线，收发共用一根，半双工

所有设备都挂在同一对总线上，通过**地址**区分从机。

---

## 二、总线电气特性

| 特性 | 说明 |
|---|---|
| 信号线 | SCL + SDA |
| 工作方式 | 同步、串行、半双工 |
| 电平 | 开漏（Open-Drain）/ 开集（Open-Collector） |
| 上拉电阻 | 典型 **4.7KΩ ~ 10KΩ**，接 VCC |
| 总线空闲 | SCL 和 SDA 都被上拉至高电平 |
| 最大从机数 | 受 7 位地址限制，理论最多 **127 个**（实际受负载电容限制） |

**为什么必须加上拉？** 因为 I2C 设备把 SCL/SDA 拉成开漏结构，只能输出低电平，高电平要靠外部上拉电阻抬上去。

---

## 三、数据位序

**MSB 先行**：先发数据最高位。

例如发送 `0xAC`（二进制 `1010 1100`），SDA 上的位序为：

```
1 → 0 → 1 → 0 → 1 → 1 → 0 → 0
MSB                 LSB
```

## 四、核心时序（PDF 原文要点）

### 1. 起始信号（Start, S）

**SCL 为高电平时，SDA 由高变低**（下降沿）。

```
SDA  ──┐    
       └────
SCL  ───────
       ↑
     Start
```

### 2. 停止信号（Stop, P）

**SCL 为高电平时，SDA 由低变高**（上升沿）。

```
SDA  ──┐    
       └────
SCL  ───────
       ↑
      Stop
```

### 3. 数据传输规则

- **SCL 为高**：SDA 必须保持稳定 → 接收方在时钟高电平期间采样
- **SCL 为低**：SDA 可以改变 → 发送方在时钟低电平期间切换数据

### 4. 应答位 ACK / NACK

每发完 8 位数据后，第 9 个时钟周期由接收方回复：

| 电平 | 含义 | 常见场景 |
|---|---|---|
| **ACK（低电平）** | 收到数据，准备好继续接收 | 正常通信 |
| **NACK（高电平）** | 不应答/接收完成/出错 | 读最后一个字节时主机发 NACK；从机不在线 |

## 五、从机地址与读写位

### 7 位地址模式

一个完整字节 = `7 位地址 + 1 位 R/W`：

```
Bit7 Bit6 Bit5 Bit4 Bit3 Bit2 Bit1 Bit0
  A6   A5   A4   A3   A2   A1   A0  R/W
```

- **R/W = 0**：主机写从机（Write）
- **R/W = 1**：主机读从机（Read）

**可分配地址数**：`2^7 = 128`，但 `0x00` 为广播地址（General Call），所以可用从机地址最多 **127 个**。

例如从机地址为 `0x10`：
- 写操作帧：`0x20`（`0x10 << 1 | 0`）
- 读操作帧：`0x21`（`0x10 << 1 | 1`）

### 10 位地址模式

当设备超过 128 个时使用：

- 第一字节：`1111 0 + A9 A8 + R/W`
- 第二字节：`A7 ~ A0`

共占用两个字节发地址，能支持到 **1024 个地址**。

## 六、完整读写时序

### I2C 写时序

1. 主机发 **起始信号 S**
2. 主机发 **7 位从机地址 + 写位（0）**
3. 等待从机 **ACK**
4. 主机发 **寄存器地址**
5. 等待从机 **ACK**
6. 主机发 **数据**（可连续多发多个字节，每字节后等 ACK）
7. 主机发 **停止信号 P**

### I2C 读时序（带重复起始，最常用）

1. 主机发 **起始信号 S**
2. 主机发 **7 位从机地址 + 写位（0）**
3. 等待从机 **ACK**
4. 主机发 **要读取的寄存器地址**
5. 等待从机 **ACK**
6. 主机发 **重复起始信号 Sr**
7. 主机发 **7 位从机地址 + 读位（1）**
8. 等待从机 **ACK**
9. 从机向主机发送 **数据**
10. 主机回复 **NACK**（读最后一个字节时）
11. 主机发 **停止信号 P**

## 七、I2C 速度等级

| 模式 | 速率 | 常见应用 |
|---|---|---|
| **标准模式（Sm）** | **100 Kbps** | EEPROM、传感器、RTC |
| **快速模式（Fm）** | **400 Kbps** | OLED、加速度计、温度传感器 |
| **快速模式+（Fm+）** | **1 Mbps** | 高吞吐外设 |
| **高速模式（Hs）** | **3.4 Mbps** | 需要高速串行总线的场景 |
| **超快速模式（UFm）** | **5 Mbps** | 只支持主机发送，应答机制不同 |

> PDF 里只列了标准模式 100KHz 和快速模式 400KHz，实际 ARM/Linux 驱动里常配这两种。

## 八、多主机与仲裁

I2C 支持**多主机**：

- 多个主机都能发起起始信号
- 仲裁规则：**SDA 低电平优先**，谁先发高电平谁退出
- 赢家继续传输，输家检测到自己发送的位和总线实际不一致就停发

这叫 **线与（Wired-AND）** 仲裁，靠开漏结构天然实现。

### 时钟拉伸（Clock Stretching）

从机如果处理不过来，可以**把 SCL 拉低**，让主机等它处理完再释放 SCL。这样低速从机不会被超速主机刷爆。

## 九、上拉电阻怎么选？

典型值 **4.7KΩ ~ 10KΩ**，但受以下因素影响：

- 总线电容越大 → 上拉电阻应越小（加速上升沿）
- 功耗敏感场景 → 上拉电阻选大点
- 速率越高 → 上拉电阻越小

经验公式（粗略估算）：

```
Rmax ≈ tr / (0.8473 × Cbus)
```

其中 `tr` 是允许的上升时间，标准模式 1000ns，快速模式 300ns；`Cbus` 是总线电容。

总线电容不能过大，标准模式限制 **400pF**，快速模式一般限制 **200pF**。

## 十、I2C vs SPI vs UART 对比

| 特性 | I2C | SPI | UART |
|---|---|---|---|
| 信号线 | 2（SCL/SDA） | 4+（SCK/MOSI/MISO/CS） | 2（TX/RX） |
| 通信方式 | 同步 | 同步 | 异步 |
| 双工 | 半双工 | 全双工 | 全双工 |
| 寻址 | 地址寻址 | 片选线寻址 | 无地址，点对点 |
| 速度 | 100K ~ 5M | 几 M ~ 几十 M | 9600 ~ 几 M |
| 多主支持 | 支持 | 不常用 | 不支持 |
| 硬件复杂度 | 简单 | 中等 | 简单 |
| 典型应用 | 传感器、EEPROM | Flash、ADC、显示屏 | 调试串口、蓝牙模块 |

## 十一、软件模拟 I2C 示例

嵌入式里经常用 GPIO 模拟 I2C：

```c
#define I2C_SCL(n)   gpio_set(SCL_PIN, n)
#define I2C_SDA(n)   gpio_set(SDA_PIN, n)
#define READ_SDA()   gpio_get(SDA_PIN)

void i2c_start(void)
{
    I2C_SDA(1);
    I2C_SCL(1);
    delay_us(5);
    I2C_SDA(0);    // SCL 高时 SDA 下降沿
    delay_us(5);
    I2C_SCL(0);
}

void i2c_stop(void)
{
    I2C_SDA(0);
    I2C_SCL(1);
    delay_us(5);
    I2C_SDA(1);    // SCL 高时 SDA 上升沿
    delay_us(5);
}
```

Linux 里则通常用 `i2c-tools`（`i2cdetect`、`i2cget`、`i2cset`）或直接读 `/dev/i2c-0`。

## 十二、常见坑点与排查

1. **没有上拉电阻或阻值太大**
   - 现象：SDA/SCL 上升沿很慢，波形圆钝，通信偶发失败
   - 解决：补 4.7KΩ 上拉，高速时换 2.2KΩ ~ 3.3KΩ

2. **地址算错**
   - 器件手册给的是 7 位地址 `0xXX`，实际发帧时要左移一位加 R/W 位
   - 例如地址 `0x50`，写帧是 `0xA0`，读帧是 `0xA1`

3. **ACK 没等到就以为失败**
   - 有些设备上电后需要延时才能响应，加几十 ms 延时试试

4. **读时序少了重复起始 Sr**
   - 先写寄存器地址再读数据，必须发重复起始，不能直接写完后停再启

5. **总线被从机拉死**
   - 某个从机异常把 SDA 或 SCL 拉低，整个总线瘫痪
   - 处理：逐个断电排查，或用 GPIO 模拟时钟把 SDA 顶出来

6. **NACK 不一定代表错误**
   - 读操作最后一个字节主机主动发 NACK 告诉从机"别发了"

7. **速率配错**
   - 从机只支持 100KHz，主机配成 400KHz 会读写出错

| 项目 | 说明 |
|---|---|
| 信号线 | SCL（时钟）、SDA（数据） |
| 总线结构 | 多主、多从、开漏、上拉 |
| 位序 | MSB 先行 |
| 地址长度 | 7 位（常用）/ 10 位 |

| 快速速率 | 400 Kbps |
| 常见器件 | EEPROM（24Cxx）、RTC（DS3231）、OLED、MPU6050、BMP280 |

## 十四、总结

I2C 最大优势就是"省线"：两根线挂一堆从机，走 PCB 布线特别方便。代价是速度不高、半双工、总线仲裁和应答机制稍微复杂。

做嵌入式 Linux 驱动时，底层基本就是：

1. 配好 GPIO 的 I2C 复用功能
2. 设置时钟频率（100K / 400K）
3. 按 S → 地址+W → ACK → 寄存器地址 → ACK → 数据 → ACK → P 的套路读写
4. 调试时先看波形对不对、ACK 有没有、地址有没有左移错
