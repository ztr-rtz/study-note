# C语言 - 函数

## 一、函数是什么

一段完成特定功能的代码块，给它起个名字就能反复调用，不用每次都从头写。

为什么要用函数：
- 避免重复写相同逻辑（复用性）
- 大问题拆成小问题逐个解决（模块化）
- 设计原则：功能越单一越好（高内聚），对外依赖越少越好（低耦合）

## 二、函数的定义

基本形式：

```c
返回值类型 函数名(参数列表)
{
    函数体;
    return 结果;
}
```

参数列表写法：`(数据类型 形参1, 数据类型 形参2, ...)`

几个要点：
1. 没写返回值类型时默认是 `int`。如果确实不需要返回值，用 `void`
2. `return` 执行后直接回到调用处，后面的代码不会再执行
3. 传参时个数要一致，类型要匹配（一致或能隐式转换）

```c
int add()
{
    int a = 20;
    int b = 10;
    int ret = a + b;
    return ret;  // 返回到调用处
}

int main()
{
    int result = add();  // () 是调用运算符
    printf("result is %d\n", result);
    return 0;
}
```

## 三、函数的声明

实际问题：函数定义不一定在调用前出现（多文件编程），编译器会警告"未声明函数"。解决办法——提前声明。

声明只告诉编译器"这个函数存在"，格式就是把函数头抄一遍：

```c
int add(int a, int b);  // 声明，只写原型
```

定义本身包含声明，所以如果定义在调用之前，声明可以省略。

## 四、函数的调用

- 调用必须用 `()`，即使没有参数也不能省略
- 传参要求：个数一致、类型匹配
- `main` 是主调函数（发起调用），`add` 是被调函数（被动执行）

```c
fun1();              // 无参调用
fun2(a, b);          // 有参调用，a/b 是实参（实际参数）
```

实参 vs 形参：实参是调用时传进去的具体值，形参是函数定义里的占位变量。

## 五、变量的作用域

### 局部变量
- 定义在函数内部（包括函数参数）
- 作用域：离定义最近的大括号范围内
- 被调函数内无法访问主调函数的局部变量

```c
int add()
{
    return a + b;  // 错！a、b 在 main 里是局部变量，这里看不到
}
int main()
{
    int a = 10;  // 局部作用域
    int b = 20;
    int c = add();
    return 0;
}
```

### 全局变量
- 定义在所有函数外面
- 作用域：整个工程的所有 `.c` 文件，任意位置都能用
- 同一作用域内标识符不能同名

## 六、变量的生命周期

### 局部变量
- 函数内部定义的变量和函数参数
- 定义时开辟内存，函数结束时回收
- 不赋初值就是随机值（垃圾数据）

### 全局变量
- 所有函数之外定义
- 程序运行前就开辟空间，程序结束后释放
- 不赋初值系统自动初始化为 0

## 七、同名变量的隐藏规则

大作用域和小作用域出现同名标识符时，小作用域里的会"隐藏"大作用域里的：

```c
int a = 10;  // 全局变量
int b = 20;

int add()
{
    return a + b;  // 用的是全局 a、b
}

int main()
{
    int a = 1;   // 局部 a 隐藏了全局 a
    int b = 3;   // 局部 b 隐藏了全局 b
    int c = add();   // c = 30（add 用全局变量）
    int d = a + b;   // d = 4（这里用的是局部变量）
    printf("c=%d, d=%d\n", c, d);
    return 0;
}
```

## 八、变量的存储类型

四种关键字修饰变量，决定变量存在内存的哪个区域：

| 存储类型 | 关键字 | 说明 | 所在区域 |
|---------|--------|------|---------|
| 自动存储 | `auto` | 局部变量默认类型，可省略。进入作用域开辟，离开时释放 | 栈区 |
| 寄存器 | `register` | 建议编译器把变量放 CPU 寄存器里，提速 | 寄存器 |
| 外部声明 | `extern` | 声明变量在其他 `.c` 文件中定义，导出供使用 | — |
| 静态存储 | `static` | 改变生命周期或限制作用域 | 数据区 |

### static 的三种用法

**1. 修饰局部变量** —— 延长生命周期到整个程序运行期，但作用域不变（仍是局部的）

```c
int fun()
{
    static int i = 0;  // 只有第一次执行时初始化，后续跳过这句
    i++;
    return i;
}
// main 里多次调用 fun()，每次返回递增的值
// 但 main 里不能直接访问 i（作用域仍在 fun 内）
```

**2. 修饰全局变量** —— 限制作用域，只能在本源文件内使用，其他 `.c` 文件看不到

**3. 修饰函数** —— 同理，限制函数只能在本源文件内调用，避免跨文件函数名冲突

### extern 跨文件示例

```c
// 1.c
extern int a;    // 声明：a 在别的文件里定义
extern int b;
extern void fun();
int main()
{
    int c = a + b;
    fun();
}

// 2.c
int a = 10;      // 定义：开辟内存
int b = 20;
void fun() { }
```

编译：`gcc 1.c 2.c`

## 九、程序运行的内存分布

`a.out` 运行后的内存布局：

| 区域 | 内容 | 特点 |
|------|------|------|
| 代码段 | 编写的代码（二进制） | 只读 |
| 数据段 | 全局变量、静态变量 | 未赋初值系统初始化为 0；可读写 |
| 堆区 | 动态内存 | 程序员手动 malloc/free，最灵活 |
| 共享区 | 库函数实现（如 printf） | — |
| 栈区 | 局部变量、函数参数、返回地址 | 自动申请/释放 |

## 十、函数的嵌套调用

函数之间可以互相调用，形成调用链：

```c
void fun3() { printf("fun3\n"); }
void fun2() { fun3(); printf("fun2\n"); }
void fun1() { fun2(); printf("fun1\n"); }

int main()
{
    fun1();
    printf("main\n");
    return 0;
}
// 输出顺序：fun3 → fun2 → fun1 → main
```

## 十一、函数参数的传递方式

### 1. 值传递

实参的值复制一份给形参，形参是副本。两个独立内存空间，修改形参不会影响实参。

```c
void swap(int a, int b)
{
    int t = a;
    a = b;
    b = t;
    printf("swap: a=%d b=%d\n", a, b);  // 交换成功
}

int main()
{
    int a = 20, b = 1;
    swap(a, b);
    printf("main: a=%d b=%d\n", a, b);  // 原值没变！
}
```

值传递只能读实参值，不能改。

### 2. 地址传递

实参传变量的地址，被调函数通过地址既能读也能改实参。

### 一维整型数组做参数

本质是地址传递。数组名传过去后降级为指针（8字节），所以必须额外传数组大小：

```c
void fill_array(int a[], int size)
{
    // a 在这里是指针，sizeof(a) = 8，不能用来算元素个数
    for (int i = 0; i < size; i++)
        a[i] = rand() % 30;
}

int main()
{
    int array[10] = {0};
    int size = sizeof(array) / sizeof(array[0]);
    fill_array(array, size);
}
```

### 一维字符数组传参

字符串约定以 `\0` 结尾，所以可以通过 `\0` 定位字符串长度，不需要额外传 size：

```c
int mystrlen(char a[])
{
    int len = 0;
    while (a[len] != '\0')
        len++;
    return len;
}

int main()
{
    char str[100] = {0};
    gets(str);
    int len = mystrlen(str);
}
```

函数版练习建议：遍历、mystrcmp、mystrcpy、mystrcat、mystrlen

## 十二、函数的递归

函数内部调用自己。本质是循环的一种形式，区别在于会持续占用栈空间。

注意点：
- 必须设置合理的退出条件
- 递归层次不要太深，否则可能栈溢出导致崩溃
- 适用场景：回溯类问题、逆向思维问题

### 阶乘示例

```c
int fun(int n)
{
    if (n == 1 || n == 0)
        return 1;
    else
        return fun(n - 1) * n;
}

int main()
{
    int ret = fun(3);  // 3! = 6
    printf("3的阶乘 %d\n", ret);
}
```

### 汉诺塔

```c
void han(int n, char a, char b, char c)
{
    if (n == 1)
        printf("%c->%c\n", a, c);
    else
    {
        han(n-1, a, c, b);     // 上层从 a 移到 b（借助 c）
        printf("%c->%c\n", a, c);  // 最底层从 a 移到 c
        han(n-1, b, a, c);     // 上层从 b 移到 c（借助 a）
    }
}

int main()
{
    han(5, 'a', 'b', 'c');
}
```

## 速查表

| 知识点 | 关键词 | 一句话要点 |
|--------|--------|-----------|
| 函数定义 | 返回值、参数列表 | 没写返回值默认 int；return 后面代码不执行 |
| 函数声明 | 原型、extern | 定义在调用前可省略声明；多文件必须声明 |
| 局部变量 | {}内定义 | 作用域在最近大括号内；不赋初值是随机值 |
| 全局变量 | 函数外定义 | 全工程可见；不赋初值系统初始化为 0 |
| 同名隐藏 | 小域隐藏大域 | 局部同名变量会遮蔽全局变量 |
| auto | 默认存储 | 局部变量默认类型，栈区，可省略 |
| register | 寄存器建议 | 请求编译器放入 CPU 寄存器 |
| extern | 跨文件声明 | 告诉编译器变量在别的 .c 里定义 |
| static(局部) | 延长生命周期 | 只初始化一次，函数结束后值不丢 |
| static(全局/函数) | 限制作用域 | 只在本源文件可见，防止跨文件冲突 |
| 值传递 | 实参副本 | 形参改不了实参 |
| 地址传递 | 传地址 | 能读能改实参 |
| 数组传参 | 降级为指针 | 整型数组需传 size；字符数组靠 \0 定长度 |
| 递归 | 自调用+退出条件 | 栈空间消耗大，层次不能太深 |
