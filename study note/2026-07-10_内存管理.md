# C语言内存管理

## 1. 程序运行时的内存布局

程序跑起来之后，操作系统会给它分配一块连续内存，按功能分成几个区段：

| 区段 | 存什么 | 特点 |
|------|--------|------|
| code（代码段） | 编译后的机器指令 | 只读，运行期间不能改 |
| data（数据段） | 有初值的全局变量、static变量 | 可读写 |
| bss | 没给初值的全局变量、static变量 | 系统自动清零 |
| rodata（只读数据段） | 字符串常量 `"hello"` 这种 | 只读 |
| heap（堆区） | 动态申请的内存 | 程序员手动 malloc/free，最大约 2.9G |
| shared（共享区） | 动态库，printf、scanf 这些函数的实现 | 共享库代码 |
| stack（栈区） | 局部变量、函数参数、返回地址 | 自动申请/释放，大约 8M |

内存地址从低到高排列：code → data → bss → heap（往上增长）→ ... → stack（往下增长）→ 共享区。

画个大概样子：

```
低地址 ┌──────────┐
       │  code    │  只读
       ├──────────┤
       │  data    │  有初值的全局/static
       ├──────────┤
       │  bss     │  无初值的全局/static
       ├──────────┤
       │  heap    │  ↓ malloc 往上增长
       │          │
       │  ......  │
       │          │
       │  stack   │  ↑ 局部变量往下增长
       ├──────────┤
       │  shared  │  共享库
高地址 └──────────┘
```

### 栈和堆的核心区别

- **栈**：系统管，进函数就分配，出函数就回收。容量小（8M），不用手动管。
- **堆**：程序员管，malloc 申请、free 释放。容量大（可增长），忘记释放就泄漏。

## 2. malloc 和 free

头文件：`#include <stdlib.h>`

### malloc — 申请堆内存

```c
void *malloc(size_t size);
```

- 参数：需要多少字节
- 返回值：堆区分配到的起始地址（void* 万能指针），失败返回 NULL
- 申请到的内存不会自动清零，里面是随机值

```c
int *p = (int *)malloc(10 * sizeof(int));  // 申请40字节，存10个int
if (p == NULL) {
    // 申请失败，处理错误
}
```

### free — 释放堆内存

```c
void free(void *ptr);
```

- 参数：传给 free 的地址必须和当时 malloc 返回的一致
- 释放后一定要 `ptr = NULL`，否则 ptr 变成野指针

```c
free(p);
p = NULL;  // 不写这行，p 就是野指针，再访问会出问题
```

### 使用流程

```
malloc → 检查返回值是否NULL → 使用内存 → free → 置NULL
```

## 3. 内存泄漏

两种常见情况：

1. **只申请不释放**：malloc 了但从来不 free，堆空间一直占着不归还。系统内存被吃光，程序最终异常退出。

2. **指针被覆盖丢失地址**：malloc 返回的地址存到指针 p 里，后来 p 又被赋了别的值，原来那块堆内存的地址就丢了，永远无法释放。

```c
int *p = (int *)malloc(40);
p = &a;  // 原来的堆地址丢了，40字节再也释放不了
```

### 防泄漏要点

- malloc 和 free 配对出现
- free 之后立刻置 NULL
- 不要覆盖还没 free 的 malloc 指针
- 堆内存的生命周期由程序员控制，从 malloc 到 free，中间不释放就一直占着

## 速查表

| 操作 | 函数/方式 | 说明 |
|------|-----------|------|
| 申请堆内存 | `malloc(size)` | 返回 void*，失败返回 NULL |
| 释放堆内存 | `free(ptr)` | ptr 必须 = malloc 返回值 |
| 释放后防野指针 | `ptr = NULL` | 释放后立即置空 |
| 栈区内存 | 局部变量 | 自动分配回收，约 8M |
| 堆区上限 | 约 2.9G | 可动态增长 |

## 易错提醒

- malloc 返回的 void* 必须强转成目标类型才能用：`(int *)malloc(...)`
- free 只能释放 malloc 的返回地址，不能传别的指针进去
- 释放后千万不要再访问那块内存，行为完全不确定
- 局部变量在栈上，函数结束就没了；malloc 在堆上，free 之前一直存在
