# ARM 汇编：立即数

## 什么是立即数？

立即数就是直接写进指令里的常数，不走内存也不走寄存器，拿来就能用。

但 ARM 的立即数不是随便写个 32 位数字就行的——它有严格的编码规则。

## 编码规则

ARM 指令 32 位固定长度，留给立即数的空间只有 12 位：

| 字段 | 位数 | 含义 |
|------|------|------|
| 8位立即数 | 8 bit | 无符号数 0~255 |
| 旋转位 | 4 bit | 把8位数循环右移 (旋转位×2) 次 |

说白了：**一个 8 位无符号数，偶数次循环右移，得到的就是合法立即数**。

不是所有 32 位数都能当立即数。比如 0x101 就不行——你找不到一个 8 位数能通过偶数旋转得到它。

## 判断示例：0xFA00

```
0xFA00 = 0x0000FA00
二进制: 0000 0000 0000 0000 1111 1010 0000 0000

0xFA = 11111010  ← 8位无符号数
循环右移 24 位 = 旋转位 12 (12×2=24)

所以 0xFA00 是合法立即数
mov r0, #0xFA00   // 合法 ✓
```

反例：0x101
```
0x101 = 0000 0000 0000 0000 0000 0001 0000 0001
找不到任何 8 位数能偶数旋转得到这个值
mov r0, #0x101    // 非法 ✗
```

遇到非法立即数怎么办？用 `ldr` 伪指令从内存加载：
```
ldr r0, =0x101    // 编译器自动把0x101放到内存池，再用ldr读出来
```

## MVN 指令

`mvn` = 取反后装入。把立即数按位取反再放进寄存器。

```
mvn r0, #0        // 0 取反 = 0xFFFFFFFF → r0 = 0xFFFFFFFF
```

这等于变相扩大了立即数范围——0 取反能得到全1，原本没法用 mov 直接装 0xFFFFFFFF。

## 速查

| 场景 | 做法 |
|------|------|
| 小常数（能通过8位偶数旋转得到的） | `mov r0, #值` |
| 大常数 / 非法立即数 | `ldr r0, =值` |
| 需要取反的值 | `mvn r0, #值` |

## 易错

1. **立即数 ≠ 任意32位数**。写 `mov r0, #0x101` 编译直接报错，别跟 x86 的 mov 混了
2. **旋转次数必须是偶数**。旋转位×2，所以实际右移位数永远是偶数
3. **mvn 是取反不是求反**。`mvn r0, #5` 得到的是 0xFFFFFFFA，不是 -5（虽然二者在补码下刚好一样，但概念不同）
4. **ldr = 是伪指令**，不是真正的 ldr。编译器会帮你处理，别跟真 ldr `ldr r0, [r1]` 搞混
